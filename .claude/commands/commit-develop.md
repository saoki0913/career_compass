# Commit to Develop

タスク完了後にdevelopブランチへ変更をコミットし、リモートにプッシュするスキル。

## 引数: $ARGUMENTS

- **コミットメッセージ**: `/commit-develop "機能を追加"` - 指定したメッセージでコミット
- **引数なし**: 対話的にコミットメッセージを生成

## 処理フロー

### 1. 現在のブランチ確認

```bash
git branch --show-current
```

- `develop`ブランチにいない場合は警告を表示
- ユーザーに確認後、必要であれば`develop`にチェックアウト

### 2. 変更内容の確認

```bash
git status
git diff --staged
git diff
```

- 変更がない場合は「コミットする変更がありません」と通知して終了
- 変更内容を要約してユーザーに表示

### 3. コミットメッセージの決定

**引数ありの場合**: `$ARGUMENTS`をコミットメッセージとして使用

**引数なしの場合**:
- 変更内容を分析して適切なコミットメッセージを提案
- 形式: `[prefix]内容`
  - `[feat]` - 新機能追加
  - `[fix]` - バグ修正
  - `[refactor]` - リファクタリング
  - `[docs]` - ドキュメント更新
  - `[style]` - コードスタイル修正
  - `[test]` - テスト追加・修正
  - `[chore]` - その他の変更

### 4. コミットとプッシュの実行

```bash
# 全ての変更をステージング
git add .

# コミット
git commit -m "コミットメッセージ"

# リモートにプッシュ
git push origin develop
```

### 5. 結果の報告

- コミットハッシュを表示
- プッシュ成功を確認
- 変更ファイル数を報告

## 注意事項

- 機密情報（.env、credentials等）がステージングされていないか確認する
- コンフリクトが発生した場合はユーザーに報告して手動解決を促す
- `main`ブランチへの直接コミットは禁止（警告を表示）
