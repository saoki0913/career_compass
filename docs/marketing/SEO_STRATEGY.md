## 1. 現状分析

### 1.1 インデックス状況（sitemap: 13URL）

**現在のsitemap.xml構成:**
- `/` - トップページ（priority: 1.0）
- `/pricing` - 料金ページ（priority: 0.8）
- `/tools` - ツール一覧（priority: 0.6）
- `/tools/es-counter` - ES文字数カウント（priority: 0.6）
- `/templates` - テンプレ集（priority: 0.6）
- `/templates/shiboudouki` - 志望動機テンプレ（priority: 0.6）
- `/templates/gakuchika-star` - ガクチカSTARテンプレ（priority: 0.6）
- `/checklists` - チェックリスト（priority: 0.5）
- `/checklists/deadline-management` - 締切管理チェックリスト（priority: 0.5）
- `/contact` - お問い合わせ（priority: 0.4）
- `/terms` - 利用規約（priority: 0.2）
- `/privacy` - プライバシーポリシー（priority: 0.2）
- `/legal` - 特定商取引法（priority: 0.2）

**問題点:**
- SEOコンテンツがわずか5ページ（tools/es-counter, templates/shiboudouki, templates/gakuchika-star, checklists/deadline-management, tools）
- ブログ基盤が存在しない
- コンテンツ更新頻度が低い（lastModified: 固定値）
- ロングテールキーワード獲得機会の損失
- ドメインオーソリティが育たない

**強み:**
- 実用的なツール・テンプレートがある（差別化要素）
- 相互リンク構造が整っている
- CTA（ログイン/無料登録）が明確
- モバイルフレンドリー（レスポンシブデザイン）

**弱み:**
- コンテンツ深度が浅い（検索上位を狙えない）
- 「就活 ES 書き方」などの広いキーワードで勝負できていない
- ブログ記事がないためロングテール獲得ゼロ
- 定期更新がない（freshness signal欠如）

### 1.3 技術SEO状況

#### ✅ 実装済み

**メタデータ:**
- `title` / `description` 各ページで適切に設定
- `metadataBase` 設定済み
- OpenGraph対応（og:type, og:title, og:description, og:locale="ja_JP"）
- Twitter Card対応（summary_large_image）
- canonical URL設定（`alternates.canonical`）

**構造化データ（JSON-LD）:**
- `SoftwareApplication` スキーマ（トップページ）
- `FAQPage` スキーマ（FAQセクション）
- Offer情報（3つの料金プラン）

**robots.txt & sitemap:**
- `/sitemap.xml` 自動生成
- `/robots.txt` 適切なallow/disallow設定
- ダッシュボード等の会員限定ページはdisallow

**パフォーマンス:**
- Next.js 16（App Router）
- 動的OG画像生成（`/opengraph-image.tsx`）
- Tailwind CSS 4（高速レンダリング）

#### ⚠️ 改善が必要

**lastModified:**
- 全ページで `new Date()` を使用しているため、実質的な更新日が反映されていない
- 推奨: ページごとに実際の更新日を管理

**Breadcrumbs:**
- パンくずリストがない → BreadcrumbList構造化データ未実装
- ユーザビリティとクローラビリティ両方に影響

**ページ速度:**
- Core Web Vitals測定未実施
- 画像最適化状況不明
- フォント読み込み最適化確認必要

**URL構造:**
- 現在の構造は良好（`/tools/`, `/templates/`, `/checklists/`）
- ただし `/blog/` セグメントが存在しない

### 3.3 制作フロー

#### ✅ 記事制作ステップ

1. **キーワード選定**（10分）
   - Google Trends、ラッコキーワード、競合調査
   - 検索意図の分類（Know/Do/Buy）

2. **競合分析**（20分）
   - 上位10記事の見出し構成を分析
   - 不足している情報を特定
   - 差別化ポイントを決定

3. **構成案作成**（30分）
   - H2/H3レベルの見出し決定
   - 各セクションの文字数配分
   - 内部リンク設計

4. **執筆**（2〜3時間）
   - 導入（200字）: 読者の悩み + 記事の価値
   - 本文（2,500字）: 見出しごとに具体的に
   - まとめ（300字）: 要点 + CTA

5. **最適化**（30分）
   - タイトル: 32文字以内、数字・記号で目立たせる
   - メタディスクリプション: 120文字、具体的なベネフィット
   - 画像: Alt属性、キャプション
   - 内部リンク: 関連記事3〜5本

6. **レビュー**（20分）
   - 誤字脱字チェック
   - 事実確認（特に統計データ）
   - リンク切れチェック
   - モバイル表示確認

// sitemap.ts
export default function sitemap(): MetadataRoute.Sitemap {

    {
      url: `${siteUrl}/`,
      lastModified: new Date('2025-02-11'),  // 最終更新日

      lastModified: post.updatedAt,
      changeFrequency: "monthly",
      priority: 0.7

  ];
}
