## 4. 紹介プログラム

### 4.1 設計概要

| 項目 | 設計 |
|------|------|
| インセンティブ | 双方向50クレジット（紹介者・被紹介者） |
| K-factor目標 | Phase 1: 0.15 → Phase 2: 0.30 → Phase 3: 0.50 |
| ティア報酬 | Bronze(1-2人): 50cr / Silver(3-5人): 75cr / Gold(6+): 100cr+バッジ |
| 発動トリガー | Aha Moment到達後 / クレジット残少時 / 高満足度検知時 |

### 4.2 不正防止ルール

- 自己紹介防止（同一メール/IPチェック）
- アクティビティ最低条件（被紹介者がAha Moment到達で報酬確定）
- レート制限（1日5紹介まで）
- 手動レビュー（異常パターン検知時）

### 4.3 ロールアウト

| Phase | 内容 | K-factor目標 |
|-------|------|-------------|
| MVP | URL紹介 + 基本報酬 | 0.15 |
| Social | SNSシェア + ティア報酬 | 0.30 |
| Gamification | リーダーボード + 限定バッジ | 0.50 |

## 6. ファネル心理学フレームワーク

| ステージ | 心理原則 | 適用例 |
|---------|---------|--------|
| Awareness | **Social Proof** | 「○○人が利用中」 |
| Acquisition | **Anchoring** | 「就活塾月3-10万 vs 就活Pass月980円」 |
| Activation | **Endowment Effect** | 無料クレジットで「もう使い始めた」感覚 |
| Revenue | **Loss Aversion** | 「残りクレジット○回、使い切る前にアップグレード」 |
| Retention | **Habit Loop** | 締切通知→タスク完了→達成感のループ |
| Referral | **Reciprocity** | 双方向50クレジット報酬 |
