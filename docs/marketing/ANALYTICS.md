## 7. 粗利課題と改善提案

### 7.1 根本課題

**ES添削（Sonnet 4.5使用）のコストが高く、年額プランで構造的赤字**

- Standard年額 ES添削集中: 粗利率 **-13%**
- Pro年額 ES添削集中: 粗利率 **0%**（損益分岐点）

### 7.2 改善提案一覧

| 優先度 | 施策 | 効果 | 対象ファイル |
|--------|------|------|-------------|
| **P0** | HyDE→Haiku変更 | ES添削コスト-9%（¥10.9→¥9.9） | `config.py:102` |
| **P0** | デッドコード削除（未使用LLMリランク） | メンテ性向上 | `llm.py`, `hybrid_search.py` |
| **P0** | キャッシュTTL 7→14日延長 | キャッシュヒット率+10% | `hybrid_search.py:45` |
| **P1** | Quick Review Mode実装（2cr, Haiku, RAGなし） | 高粗利商品追加（Std 48%） | `es_review.py` |
| **P1** | ES添削最低3cr引き上げ検討 | 赤字解消 | `schema.ts` |
| **P2** | 年額割引率 16%→12% | Std年額粗利 -13%→-2% | Stripe設定 |
| **P2** | RAGコスト分離計測 | コスト可視化 | `hybrid_search.py` |

### 7.3 施策適用後の改善予測

| 指標 | 現在 | HyDE→Haiku後 | +割引率12%後 |

| Std月額 ES粗利率 | 5% | 14% | 14% |
| Std年額 ES粗利率 | -13% | -4% | -2%（ほぼ損益分岐） |
| Pro月額 ES粗利率 | 17% | 24% | 24% |
| Pro年額 ES粗利率 | 0% | +7% | +10% |

## 11. 前提条件・注意事項

- トークン数は日本語推定（1文字 ≈ 2-3トークン）
- 為替: ¥1 ≈ $0.0065（2026年2月）
- ES添削クレジット計算: `max(2, ceil(charCount / 800))`、上限5
- Cross-encoder（ローカル）はAPIコスト不要
- インフラコスト（Supabase, Vercel, Railway等）は本分析に含まない
- 季節性（3-4月ピーク、8-9月閑散期）は考慮していない
