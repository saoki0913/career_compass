## 第1章: 現状分析

### 1.1 現在のLP構成

LP のルートは `src/app/(marketing)/page.tsx`。以下の8セクションで構成される。

```
1. LandingHeader        — 固定ヘッダー（ロゴ + ナビ + 認証状態別CTA）
2. HeroSection          — メインコピー + スクリーンショット3枚 + 信頼バッジ
3. ProductShowcase      — 3つの価値カード + 詳細2セクション
4. HowItWorksSection    — 3ステップの使い方
5. PricingSection       — Free / Standard / Pro カード（月額のみ）
6. FAQSection           — 7問（アコーディオン、左右分割レイアウト）
7. CTASectionVariant    — 感情訴求クロージングCTA
8. Footer               — 4カラム（ブランド / Product / Public Pages / Legal）
```

### 1.3 現在の強み

| 項目 | 詳細 |
|------|------|
| コピーの一貫性 | 「統合」「途中から始められる」「低ハードル」のメッセージが全セクションで一貫 |
| 信頼バッジ | trustPoints がヒーローと最終CTAの両方に配置 |
| ビジュアル品質 | ScrollReveal アニメーション、ScreenPreview、グラデーション背景、洗練されたカードUI |
| SEO基盤 | FAQPage JSON-LD、WebSite + SoftwareApplication の JSON-LD graph、適切な metadata |
| 認証状態対応 | LandingPrimaryAction がゲスト/ログイン済み/未認証の3状態に自動対応 |
| レスポンシブ | モバイルファーストのグリッドレイアウト、モバイルハンバーガーメニュー |

### 1.4 現在の弱み（ギャップ分析）

| カテゴリ | 不足要素 | 重要度 |
|----------|----------|--------|
| ソーシャルプルーフ | ユーザー数、利用者の声、満足度、大学名、内定先が一切ない | 最高 |
| 問題提起/痛みの増幅 | Before/After、就活の不安を具体的に描写するセクションがない | 最高 |
| 競合比較 | 就活塾 vs 手運用 vs 就活Pass の比較が不在 | 高 |
| 変化のイメージ | 導入後の具体像、成果の可視化がない | 高 |
| 面接対策の訴求 | 実装済みの主要機能だがLPでほぼ言及されていない | 中 |
| 緊急性 | 就活シーズンに合わせた時間軸訴求がない | 中 |
| インタラクティブ要素 | ライブデモ、操作体験がない | 中 |
| スティッキーCTA | モバイルでの固定CTAバーがない | 高 |
| CTA接触頻度 | ヒーローと最終CTAの2箇所のみ。中間CTAがない | 中 |
