# コンバージョンファネル設計

## 概要

就活Pass（Career Compass）のコンバージョンファネルを、AARRRフレームワーク（Acquisition, Activation, Retention, Revenue, Referral）と心理学的最適化の視点から設計する。

---

## 1. AARRR フレームワーク概要

```
Awareness（認知） → Acquisition（獲得） → Activation（活性化） → Revenue（収益化） → Retention（継続） → Referral（紹介）
```

### 現状の課題
1. **Social Proof欠如**: TestimonialsSectionが実際のユーザーの声ではなく、機能ハイライト
2. **Aha Moment未定義**: 初回体験で「価値を実感できた瞬間」が明確でない
3. **ファネル計測基盤あり、戦略ドキュメントなし**: GA4で100+イベント計測しているが、改善戦略が未文書化
4. **季節性未対応**: 就活シーズン（3〜6月、10〜12月、1月追い込み）に応じたメッセージング変化が未設計

---

## 2. Awareness（認知）

### 目的
ターゲットユーザーに就活Passの存在を知ってもらい、興味を持ってもらう。

### ターゲットペルソナ
- 不安先行型（美咲）: Instagram、X、友人の口コミ
- 効率追求型（翔太）: Google検索、Qiita、note
- 地方ハンデ型（あかり）: Instagram、LINE、友人の口コミ
- 直前追い込み型（大樹）: X、YouTube、友人の口コミ
- コスパ重視型（結衣）: Google検索、比較サイト、口コミサイト

### チャネル別認知獲得戦略

#### 1. SEO（Search Engine Optimization）
**対象ペルソナ**: 効率追求型、コスパ重視型

**キーワード戦略**:
- **ブランドキーワード**: 「就活Pass」「Career Compass」
- **機能キーワード**: 「ES添削 AI」「就活 締切管理」「企業研究 ツール」
- **比較キーワード**: 「就活塾 比較」「ES添削 無料」「就活ツール おすすめ」
- **ロングテールキーワード**: 「ES 書き方 分からない」「就活 何から始める」「締切 管理 アプリ」

**コンテンツ戦略**:
- ブログ記事: 「ESの書き方完全ガイド」「就活スケジュール管理術」
- 比較記事: 「就活塾 vs 就活アプリ 徹底比較」
- ガイド: 「業界別志望動機の書き方」

**KPI**:
- オーガニック流入数: 月1000セッション（3ヶ月後）
- 上位表示キーワード数: 10キーワード（6ヶ月後）

#### 2. SNS広告（Instagram, X, YouTube）
**対象ペルソナ**: 不安先行型、地方ハンデ型、直前追い込み型

**クリエイティブ戦略**:
- **静止画**: Before/After形式（「ESの質が劇的UP」）
- **動画**: 15秒の機能紹介（「30秒で登録、今日から使える」）
- **カルーセル**: 3つの機能紹介（添削・締切・企業研究）

**メッセージング**:
- 不安先行型向け: 「締切を落とさず、ESの質を上げて、自信を持って就活できる」
- 地方ハンデ型向け: 「地方でも、都市部と同じ就活サポートが受けられる」
- 直前追い込み型向け: 「今からでも間に合う。30秒で登録、即日添削」

**ターゲティング**:
- 年齢: 20〜23歳
- 興味関心: 就活、キャリア、大学生活
- 行動: 就活関連ページの閲覧

**KPI**:
- CTR: 1.5%以上
- CPC: 50円以下
- CVR（登録率）: 5%以上

#### 3. 友人の口コミ（Word of Mouth）
**対象ペルソナ**: 全ペルソナ

**戦略**:
- 紹介プログラム: 紹介者・被紹介者双方に50クレジットプレゼント
- SNSシェア機能: 「ES添削結果をシェア」「内定報告をシェア」
- バイラルループ: 紹介→登録→活用→紹介の循環

**KPI**:
- 紹介経由の登録率: 10%（6ヶ月後）
- バイラル係数（K-factor）: 0.3以上（1人が0.3人を紹介）

#### 4. 大学との提携
**対象ペルソナ**: 全ペルソナ

**戦略**:
- キャリアセンターとの提携: 就活Passを公式推薦ツールとして紹介
- 学生団体との提携: 就活イベントでのブース出展、チラシ配布
- 学割プログラム: 大学ドメインのメールアドレスで登録すると初回50クレジットプレゼント

**KPI**:
- 提携大学数: 10校（6ヶ月後）
- 大学経由の登録数: 月100件（6ヶ月後）

## 3. Acquisition（獲得）

### 目的
ランディングページ（LP）を最適化し、訪問者を登録ユーザーに転換する。

### 現状のLP構成
1. **HeroSection**: ヘッドライン、サブヘッド、CTA、スクリーンショットカルーセル
2. **SocialProofSection**: 信頼バッジ（「無料で始められる」「30秒で登録」）
3. **FeaturesSection**: 3つの主要機能（添削・締切・企業研究）
4. **HowItWorksSection**: 3ステップ（登録→企業登録→ES添削）
5. **ComparisonSection**: 就活Pass vs 無料ツール vs 就活塾
6. **TestimonialsSection**: 機能ハイライト（実際のユーザーの声ではない）← **要改善**
7. **PricingSection**: Free/Standard/Pro 3プラン
8. **FAQSection**: 6つのFAQ
9. **CTASection**: 最終CTA

### LP最適化戦略

#### 1. HeroSection最適化

**現状**:
- ヘッドライン: 「内定に、最短で近づく就活AI。」
- サブヘッド: 「ES添削 × 締切管理 × 企業研究 — 3つのAI機能で就活の不安をなくす。」
- CTA: 「無料で始める」「ゲストで試す」
- スクリーンショットカルーセル: 4つの機能画面

**心理学的最適化**:
- **Anchoring（アンカリング）**: ヘッドラインで「就活塾30,000円〜」を提示し、後で「月980円」を提示してコントラスト効果を生む
- **Loss Aversion（損失回避）**: 「締切を落とさない」「ESの機会を逃さない」をサブヘッドに追加
- **Social Proof（社会的証明）**: 「○○人が利用中」「○○社の企業情報を管理」を追加（実績数字を収集後）
- **Cognitive Load削減**: ヘッドラインを1行に簡潔化（「内定に近づく、就活AI。」）

**改善案**:
```
ヘッドライン: 「締切を落とさず、ESの質を上げて、内定に近づく。」
サブヘッド: 「就活塾は月30,000円〜。就活Passなら月980円で、添削・締切・企業研究が使い放題。」
社会的証明: 「すでに○○人が利用中」（数字未収集、将来追加）
CTA: 「無料で始める（カード不要）」「ゲストで試す」
```

#### 2. SocialProofSection強化

**現状**:
- 信頼バッジ: 「無料で始められる」「30秒で登録」
- Google認証ロゴ

**心理学的最適化**:
- **Authority（権威）**: 提携大学のロゴ、メディア掲載実績（将来）
- **Social Proof（社会的証明）**: ユーザー数、添削回数、企業登録数
- **Bandwagon Effect（バンドワゴン効果）**: 「○○大学の学生が利用中」

**改善案**:
```
- 「累計○○人が利用中」
- 「累計○○回のES添削を実施」
- 「○○社の企業情報を管理」
- 提携大学ロゴ（将来）
- メディア掲載ロゴ（将来）
```

#### 4. ComparisonSection強化

**現状**:
- 就活Pass vs 無料ツール vs 就活塾の比較表

**心理学的最適化**:

#### 5. PricingSection最適化

**現状**:
- Free: 月0円、30クレジット
- Standard: 月980円、300クレジット（「一番人気」バッジ）
- Pro: 月2,980円、800クレジット

**心理学的最適化**:
- **Anchoring（アンカリング）**: Proを先に提示し、Standardを「お得」に見せる
- **Decoy Effect（デコイ効果）**: ProとStandardの価格差を調整し、Standardを最適選択に
- **Framing（フレーミング）**: 「1日33円」と表示してコストを小さく見せる
- **Charm Pricing（チャームプライシング）**: 980円（既に実装）

**改善案**:
```
提示順序: Pro → Standard（一番人気） → Free
Standard: 「1日たった33円」を強調
Pro: 「本気で内定を狙う人向け」
Free: 「まずはお試し」
```

### ゲストモード → 登録フロー

**現状**:

**心理学的最適化**:
- **Foot-in-the-Door（段階的コミットメント）**: ゲストで小さなコミットメント（企業登録）→ログインで大きなコミットメント
- **Endowment Effect（保有効果）**: ゲストで作成したデータが「自分のもの」に感じられ、ログインして引き継ぎたくなる
- **Loss Aversion（損失回避）**: 「ゲストデータは7日で削除されます」警告

**改善案**:
```
ゲスト利用中:
- 「ゲストデータは7日で削除されます。ログインしてデータを保存しましょう」バナー表示（控えめ）
- 機能制限時: 「この機能を使うにはログインが必要です」（1回のみ表示）

ゲスト→ログイン移行時:
- 「あなたのデータを引き継ぎます」メッセージ
- 「30秒で完了。カード不要」
```

## 4. Activation（活性化）

### 目的
登録ユーザーに「Aha Moment（価値を実感できた瞬間）」を体験してもらい、アクティブユーザーに転換する。

### Aha Moment定義（最重要）

#### 現状の課題
- Aha Momentが明確に定義されていない
- 初回体験の最適化が不十分

#### Aha Moment候補

##### 候補1: 「企業情報が自動で取れた！」
- **トリガー**: 企業登録 → 企業情報抽出成功
- **心理**: 「手作業で調べる時間が省ける」「これは便利だ」
- **指標**: 企業登録から情報抽出成功までの時間、成功率

##### 候補2: 「AI添削、具体的で役立つ！」
- **トリガー**: ES添削実行 → 具体的な改善提案を受け取る
- **心理**: 「無料ツールと全然違う」「これなら質が上がる」
- **指標**: ES添削実行率、添削結果の閲覧時間

##### 候補3: 「今日やるべきことが分かった！」
- **トリガー**: ダッシュボード表示 → 「今日の最重要タスク」を確認
- **心理**: 「迷わない」「何をすればいいか明確」
- **指標**: ダッシュボード訪問回数、タスク完了率

#### 推奨Aha Moment
**「企業情報が自動で取れた！」 + 「AI添削、具体的で役立つ！」**

理由:
- 2つの主要機能を体験できる
- 即座に価値を実感できる（企業情報抽出は30秒、AI添削は2〜3分）
- 心理的インパクトが大きい

### オンボーディング最適化

#### 現状のオンボーディングフロー
```
登録 → プラン選択 → 3ステップ質問（大学情報、志望業界、ガクチカ） → ダッシュボード
```

**課題**:
- 3ステップ質問が面倒に感じる
- スキップされることが多い（想定）
- Aha Momentまでの距離が遠い

#### 最適化戦略

##### 1. Progressive Disclosure（段階的開示）
- 初回は最小限の情報のみ入力
- 後から追加入力を促す

##### 2. Goal-Gradient Effect（目標勾配効果）
- プログレスバーを表示し、「あと2ステップ」を明示
- 完了に近づくほど加速する心理を活用

##### 3. Aha Momentへの最短ルート設計
```
改善案:
登録 → プラン選択 → 「企業を1社登録してみましょう」 → 企業情報自動抽出（Aha Moment 1）
  → 「ESを1つ作成してみましょう」 → ES添削実行（Aha Moment 2）
  → 「あなたについて教えてください」（3ステップ質問）
```

##### 4. Activation Checklist（アクティベーションチェックリスト）
ダッシュボードに「はじめ方ガイド」を表示:
- [ ] 企業を1社登録する
- [ ] ES添削を1回実行する
- [ ] 締切を1件承認する
- [ ] プロフィールを完成させる

**心理トリガー**:
- **Zeigarnik Effect（ツァイガルニク効果）**: 未完了タスクは記憶に残りやすい
- **Goal-Gradient Effect**: チェックリストが埋まるほど完了したくなる

#### KPI

| KPI | 現状 | 目標（3ヶ月後） | 目標（6ヶ月後） |
|-----|------|----------------|----------------|
| オンボーディング完了率 | - | 60% | 70% |
| Aha Moment到達率（企業情報抽出） | - | 50% | 65% |
| Aha Moment到達率（ES添削） | - | 40% | 55% |
| 登録後7日間のアクティブ率 | - | 50% | 60% |
| 登録後30日間のアクティブ率 | - | 30% | 40% |

## 5. Revenue（収益化）

### 目的
無料ユーザーを有料ユーザーに転換し、収益を最大化する。

### 現状の有料プラン
- **Standard**: 月980円、300クレジット
- **Pro**: 月2,980円、800クレジット

### Free → Standard/Pro 転換トリガー

#### トリガー1: クレジット不足
- **タイミング**: 無料クレジット（30）が残り5以下になったとき
- **メッセージ**: 「クレジットが不足しています。Standardプラン（月980円、300クレジット）にアップグレードしませんか？」
- **心理トリガー**: Loss Aversion（損失回避）「このままでは添削を受けられない」

#### トリガー2: 企業登録上限
- **タイミング**: 企業登録が5社に達したとき
- **メッセージ**: 「企業登録の上限に達しました。Standardプラン（無制限）にアップグレードしませんか？」
- **心理トリガー**: Loss Aversion「新しい企業を登録できない」

#### トリガー3: 機能制限
- **タイミング**: 有料機能（リライト3本、スタイル8種）を利用しようとしたとき
- **メッセージ**: 「この機能はStandardプラン以上で利用できます」
- **心理トリガー**: Curiosity Gap（好奇心ギャップ）「他の機能を試したい」

#### トリガー4: 成果実感後
- **タイミング**: ES添削を3回以上実行し、書類通過の報告があったとき
- **メッセージ**: 「おめでとうございます！ES添削の成果が出ていますね。Standardプランでさらに添削回数を増やしませんか？」
- **心理トリガー**: Confirmation Bias（確認バイアス）「役立っている」という実感

#### トリガー5: 季節性（締切ラッシュ）
- **タイミング**: 就活シーズン（3〜6月、10〜12月）
- **メッセージ**: 「締切ラッシュの時期です。Standardプランで応募企業を増やしませんか？」
- **心理トリガー**: Urgency（緊急性）、Scarcity（希少性）「今がチャンス」

### アップセルUI設計

#### 1. クレジット不足通知
```
トースト通知:
「クレジットが残り5になりました。Standardプラン（月980円、300クレジット）にアップグレードしませんか？」
[今すぐアップグレード] [後で]
```

#### 2. プラン比較モーダル
```
機能制限時にモーダル表示:
「この機能はStandardプラン以上で利用できます」

比較表:
| 機能 | Free | Standard | Pro |
|------|------|----------|-----|
| 月次クレジット | 30 | 300 | 800 |
| 企業登録 | 5社 | 無制限 | 無制限 |
| リライト | 1本 | 3本 | 3本 |
| 添削スタイル | 3種 | 8種 | 8種 |

[Standardにアップグレード（月980円）] [Proにアップグレード（月2,980円）]
```

#### 3. ダッシュボードバナー
```
クレジット残高表示の下に:
「あと270クレジット使えます。Standardプランで月10回以上添削できます。」
[詳細を見る]
```

### 心理学的価格設計

#### 1. Anchoring（アンカリング）
- **戦略**: 就活塾の料金（月30,000円〜100,000円）を先に提示
- **効果**: Standardの980円が「圧倒的に安い」と感じる

#### 2. Decoy Effect（デコイ効果）
- **戦略**: Proプランをデコイとして配置
- **効果**: Standardが「最もコスパが良い」と感じる

#### 3. Framing（フレーミング）
- **戦略**: 「月980円」を「1日33円」と表示
- **効果**: コストが小さく感じる

#### 4. Mental Accounting（メンタルアカウンティング）
- **戦略**: 「交通費1回分以下」「カフェのコーヒー3杯分」と比較
- **効果**: 「大した出費ではない」と感じる

#### 5. Zero-Price Effect（ゼロ価格効果）

### 習慣化ループ設計

#### 習慣化の公式（BJ Fogg Behavior Model）
```
Behavior = Motivation × Ability × Prompt
```

- **時間を短縮**: 添削は2〜3分で完了
- **ガイドを提供**: 使い方ガイド、チュートリアル

#### 3. Prompt（きっかけ）を設計
- **時間ベース**: 毎日朝9時に「今日のタスク」通知
- **イベントベース**: 締切3日前に通知、クレジット不足時に通知
- **位置ベース**: アプリ起動時にダッシュボード表示

## 9. 心理学的最適化

### 認知バイアスの活用

#### 1. Anchoring（アンカリング）
**適用箇所**: PricingSection、ComparisonSection

**戦略**:
- 就活塾の料金（月30,000円〜100,000円）を先に提示
- Proプランを先に表示し、Standardを「お得」に見せる

**実装**:
```
ComparisonSection:
就活塾: 月30,000円〜100,000円 ← アンカー
就活Pass: 月980円 ← 「圧倒的に安い」と感じる

PricingSection:
Pro: 月2,980円 ← アンカー
Standard: 月980円 ← 「お得」と感じる
Free: 月0円
```

#### 2. Decoy Effect（デコイ効果）
**適用箇所**: PricingSection

**戦略**:
- Proプランをデコイとして配置
- StandardとProの価格差を調整し、Standardを最適選択に

**実装**:
```
Free: 月0円、30クレジット
Standard: 月980円、300クレジット ← 「一番人気」バッジ
Pro: 月2,980円、800クレジット ← デコイ（価格差が大きい）

計算:
Standard: 1クレ3.2円
Pro: 1クレ3.7円 ← Standardより割高に見える（意図的）
```

#### 3. Loss Aversion（損失回避）
**適用箇所**: HeroSection、CTASection、通知

**戦略**:
- 「締切を落とさない」「ESの機会を逃さない」を強調
- ゲストデータ削除警告

**実装**:
```
HeroSection:
「締切を落とさず、ESの質を上げて、内定に近づく。」

CTASection:
「就活シーズンは待ってくれない。今日から、就活の準備を始めよう。」

ゲストモード:
「ゲストデータは7日で削除されます。ログインしてデータを保存しましょう」
```

#### 4. Social Proof（社会的証明）
**適用箇所**: HeroSection、SocialProofSection、TestimonialsSection

**戦略**:
- ユーザー数、添削回数、企業登録数を表示
- 実際のユーザーの声を掲載
- 提携大学ロゴ、メディア掲載実績を表示

**実装**:
```
HeroSection:
「すでに○○人が利用中」（数字未収集、将来追加）

SocialProofSection:
「累計○○人が利用中」
「累計○○回のES添削を実施」
「○○社の企業情報を管理」

TestimonialsSection:
実際のユーザーの声（名前、大学、成果）
```

### 信頼構築要素

#### 1. Transparency（透明性）
- **料金の透明性**: 追加料金なし、明朗会計
- **データの透明性**: 企業情報抽出の根拠URL表示、信頼度バッジ
- **プロセスの透明性**: ES添削の評価基準を公開

#### 2. Security（セキュリティ）
- **Google認証**: 「Googleアカウントで安全にログイン」
- **データ保護**: 「通信はすべて暗号化」
- **プライバシー**: 「学習には使用されません」

#### 3. Support（サポート）
- **FAQ**: 6つのFAQを掲載（さらに拡充）
- **お問い合わせ**: メール、チャットサポート
- **使い方ガイド**: チュートリアル、動画ガイド

#### 4. Authority（権威）
- **提携大学**: 大学ロゴを掲載（将来）
- **メディア掲載**: 「○○メディアに掲載されました」（将来）
