## 2. 機能別コスト分析

### 2.1 ES添削（最も高コスト機能）

**トークン使用量（推定）**:
- Input: ~3,000 tokens（ES本文 + システムプロンプト + RAGコンテキスト）
- Output: ~1,500 tokens（スコア + 改善点 + リライト3本）

**APIコスト計算**:
```
Input:  3,000 × $3.00 / 1M = $0.009
Output: 1,500 × $15.00 / 1M = $0.0225
合計: $0.0315 ≈ 4.7円/回
```

**クレジット消費**: `ceil(文字数 / 800)`, 上限5

| ES文字数 | クレジット | 収益(Standard) | コスト | 利益率 |
|----------|-----------|----------------|--------|--------|
| 400字 | 1 | 3.27円 | 4.7円 | **-44%** |
| 800字 | 1 | 3.27円 | 4.7円 | **-44%** |
| 1,200字 | 2 | 6.54円 | 4.7円 | +28% |
| 2,400字 | 3 | 9.81円 | 4.7円 | +52% |
| 4,000字 | 5 | 16.35円 | 4.7円 | +71% |

**問題点**: 800字以下のES添削は赤字になる可能性が高い

### 2.2 企業情報取得（低コスト機能）

**トークン使用量**:
- Haiku: ~2,000 input, ~500 output
- Embeddings: ~50ページ × ~500 tokens/page = 25,000 tokens

**APIコスト計算**:
```
Haiku:     (2,000 × $1 + 500 × $5) / 1M = $0.0045
Embedding: 25,000 × $0.02 / 1M = $0.0005
合計: ~$0.005 ≈ 0.75円/回
```

**クレジット消費**: 1（無料枠3回/日超過時）

| 条件 | 収益 | コスト | 利益率 |
|------|------|--------|--------|
| 有料消費 | 3.27円 | 0.75円 | **+336%** |
| 無料枠 | 0円 | 0.75円 | - |

### 2.3 ガクチカ深掘り（中コスト機能）

**トークン使用量**（5問あたり）:
- Input: ~4,000 tokens（会話履歴 + システムプロンプト）
- Output: ~1,500 tokens（質問 + フィードバック）

**APIコスト計算** (**2026-02-11更新**: 実際のモデルはClaude Haiku 4.5):
```
Input:  4,000 × $1.00 / 1M = $0.004
Output: 1,500 × $5.00 / 1M = $0.0075
合計: $0.0115 ≈ 1.8円/5問
```

**クレジット消費**: 1クレジット/5問

| 条件 | 収益 | コスト | 利益率 |
|------|------|--------|--------|
| Standard | 3.27円 | 1.8円 | **+82%** ◎ |
| Pro | 3.73円 | 1.8円 | **+107%** ◎ |

### 2.4 志望動機作成（中コスト機能）

ガクチカと同様のコスト構造 + 下書き生成で追加1クレジット

## 3. 月間収支シミュレーション

### 3.1 想定ユーザー行動パターン

**アクティブユーザー（Standard）の月間利用想定**:
- ES添削: 15回（平均800字 = 15クレジット）
- 企業取得: 30回（無料枠90回 + 有料0回 = 0クレジット）
- ガクチカ: 20問（4クレジット）
- 合計: 19クレジット消費

**コスト計算**:
```
ES添削:    15回 × 4.7円 = 70.5円
企業取得:  30回 × 0.75円 = 22.5円
ガクチカ:  4回 × 5.2円 = 20.8円
合計コスト: 113.8円
```

**収益**: 980円/月
**粗利益**: 980 - 114 = **866円 (88%)**

### 3.2 ヘビーユーザー（Pro）の月間利用想定

**ヘビーユーザーの月間利用想定**:
- ES添削: 50回（平均1,200字 = 100クレジット）
- 企業取得: 100回（無料枠90回 + 有料10回 = 10クレジット）
- ガクチカ: 50問（10クレジット）
- 志望動機: 20問 + 下書き4回（8クレジット）
- 合計: 128クレジット消費

**コスト計算**:
```
ES添削:    50回 × 4.7円 = 235円
企業取得:  100回 × 0.75円 = 75円
ガクチカ:  10回 × 5.2円 = 52円
志望動機:  8回 × 5.2円 = 41.6円
合計コスト: 403.6円
```

**収益**: 2,980円/月
**粗利益**: 2,980 - 404 = **2,576円 (86%)**

## 4. 価格設定の評価

### 4.1 現状の強み

1. **高い粗利益率**: 典型的ユーザーで86-88%
2. **無料枠の適切な設計**: 企業取得3回/日で体験を担保
3. **成功時のみ消費**: ユーザー体験◎、コスト予測可能

### 4.2 現状の課題

1. **短文ES添削の赤字リスク**: 800字以下で損失
2. **クレジット単価の逆転現象**: Proの方が単価が高い
3. **ガクチカ/志望動機**: ~~構造的に赤字~~ → Haiku化により粗利率60-82%に改善済み ✅

### 4.3 競合価格帯との比較

| サービス | 月額 | 主要機能 |
|----------|------|----------|
| 就活Pass Standard | 980円 | ES添削 + 進捗管理 |
| 就活Pass Pro | 2,980円 | 上記 + 高RAG上限 |
| 就活塾 | 10,000-50,000円 | 人力添削 + 相談 |
| ES添削サービス（他社） | 500-2,000円/回 | 単発添削 |

**ポジショニング**: 就活塾の1/10以下で「安価に、迷わず」を実現

## 5. 改善提案

### 5.1 即時対応（コスト最適化）

#### A. ES添削の最小クレジット引き上げ
```
現在: ceil(文字数 / 800), 上限5
提案: max(2, ceil(文字数 / 800)), 上限5
```
- 短文添削の赤字を解消
- 400字ESでも2クレジット（6.54円）で黒字化

#### B. ガクチカ/志望動機のモデル変更 ✅ **実装済み**
```
実装済み: Claude Haiku 4.5
(backend/app/config.py: model_gakuchika="claude-haiku", model_motivation="claude-haiku")
```
- コスト改善: 5.2円 → 1.8円/5問 (-65%)
- 品質面: Haiku 4.5は対話型タスクで十分な品質を維持

### 5.2 中期対応（価値向上）

#### C. Pro価格の引き下げまたは価値追加
```
選択肢1: Pro 2,480円に値下げ（クレジット単価をStandard以下に）
選択肢2: Pro限定機能追加（業界別攻略Tips、AI面接練習など）
```

#### D. 年間プラン導入
```
Standard年間: 9,800円（2ヶ月無料）
Pro年間: 29,800円（2ヶ月無料）
```
- LTV向上、解約率低減

### 5.3 長期検討（価値メトリクス見直し）

#### E. クレジット制から機能制へ
```
現在: クレジット消費型（使用量課金）
検討: 機能制限型（回数上限）
```

| プラン | ES添削 | 企業登録 | ガクチカ |
|--------|--------|----------|----------|
| Free | 5回/月 | 5社 | 2素材 |
| Standard | 30回/月 | 無制限 | 10素材 |
| Pro | 無制限 | 無制限 | 20素材 |

- メリット: ユーザーにわかりやすい、予測可能なコスト
- デメリット: ヘビーユーザーで赤字リスク

## 6. 更新版アクションプラン

**注**: 2026-02-11更新。CREDIT_PROFITABILITY_ANALYSIS.mdの最新分析を反映。

### Critical Priority: 年額プラン赤字対策

#### 6-1. RAG HyDEモデルをHaikuに変更（即時実装推奨）

**ファイル**: `backend/app/config.py`

```python
# 変更前
model_rag_hyde: str = "claude-sonnet"  # MODEL_RAG_HYDE

# 変更後
model_rag_hyde: str = "claude-haiku"  # MODEL_RAG_HYDE
```

**コスト効果**:
- ES添削APIコスト: ¥10.9 → ¥9.9（-9%削減）
- Standard年額 ES添削粗利率: -13% → -4%（+9pp改善）
- Pro年額 ES添削粗利率: 0% → +7%（+7pp改善）

**品質リスク**: HyDEは200-400文字の短文生成なので、Haikuでも品質維持が期待される。実装後に品質検証推奨。

#### 6-2. 年額割引率の調整（検討推奨）

**選択肢A: 割引率を16%→12%に縮小**
```
現在: Standard ¥9,800/年, Pro ¥29,800/年
推奨: Standard ¥10,400/年, Pro ¥31,600/年
```

**効果**:
- HyDE変更と組み合わせで、Standard年額の最悪ケースが黒字化
- Pro年額の粗利率が0% → +10%に改善

**選択肢B: 年額割引率維持（16%）、HyDE変更のみ**
- Standard年額の最悪ケースは依然赤字（-¥28/月）
- 典型利用パターンでは黒字維持

**推奨**: 選択肢A（割引率12%）が最も安全。ただし、初年度は16%維持してユーザー獲得を優先し、Year 2から調整する戦略も検討可能。

### High Priority: コスト最適化

#### 6-3. ES添削の最小クレジット引き上げ（完了済み）

**ファイル**: `src/lib/credits/cost.ts`

```typescript
// 既に実装済み
export function calculateESReviewCost(charCount: number): number {
  return Math.min(5, Math.max(2, Math.ceil(charCount / 800)));
}
```

**状況**: コード上既に実装済み。この施策により短文ES添削の赤字は解消済み。

#### 6-4. デッドコードの削除

**対象ファイル**:
- `backend/app/config.py:103` (model_rag_rerank 設定)
- `backend/app/utils/hybrid_search.py:707` (rerank_results_with_llm 関数)

**理由**: LLMリランキングは使用されておらず、ローカルcross-encoderで代替済み。

### Medium Priority: 中期施策

#### 6-5. クイックレビューモード導入（要価格設計）

**現状の課題**: 1クレジット設定では赤字または薄利

**推奨アプローチ**:
- Option 1: Quick Review 2クレジット（粗利率 Standard 48%、Pro 57%）
- Option 2: RAGなし簡易版 1クレジット、APIコスト ¥1.2（粗利率 Standard 63%、Pro 68%）

#### 6-6. クエリ拡張キャッシュ改善

```
現在: ハッシュベース完全一致、ヒット率25%推定
目標: コサイン類似度0.95以上でキャッシュヒット、ヒット率50%
節約: ES添削1回あたり¥0.5程度
```

## 8. 重要な発見と次のステップ

### 8.1 クレジット数の不一致解消

**発見**: PRICING_ANALYSIS.md（旧版）では Standard 300クレジットと記載していたが、CREDIT_PROFITABILITY_ANALYSIS.md（旧版）では誤って100クレジットと記載。

**修正**: 両ドキュメントでStandard 300クレジットに統一（コード上の正しい値に一致）。

### 8.2 3x ES添削機能の誤解

**発見**: 旧分析では3x ES添削が主要な赤字要因として分析されていたが、**実際にはこの機能は使用されていない**。

**修正**: 全ての分析から3x ES添削を除外。実際の赤字リスクは「年額プランでのES添削集中利用」。

### 8.3 年額プランの収益性リスク

**新たな発見**:
- Standard年額 (¥2.72/credit) はES添削集中利用で-13%の赤字
- Pro年額 (¥3.10/credit) はES添削集中利用で損益分岐点
- 16%割引により、ES添削のユニットエコノミクスが悪化

**推奨対策**: HyDE→Haiku変更 + 年額割引率12%調整で黒字化

### 8.4 次のステップ

1. **即時実装**: RAG HyDEモデルをHaikuに変更（ES添削コスト-9%削減）
2. **戦略判断**: 年額割引率を16%→12%に調整するか、16%維持でリスク許容するか
3. **モニタリング**: 年額プラン実装後、実際の利用パターンとAPI costを追跡
4. **A/Bテスト**: クイックレビューモードの需要検証（2cr vs RAGなし1cr）
