# 就活Pass LP デザインブリーフ（Claude Design 入力用）

> **目的**: Claude Design で LP（ランディングページ）のデザインモック / 画像を生成するための、自己完結型プロダクト & デザイン仕様書。
> **対象読者**: AI デザインツール。コードや CSS 変数は含めず、hex 値・日本語コピー・レイアウト指示で完結する。
> **実装リファレンス**: [`docs/marketing/LP.md`](./LP.md)

---

## 1. プロダクト概要

### 1.1 基本情報

| 項目 | 内容 |
|------|------|
| アプリ名 | **就活Pass**（読み: シューパス / 英字: ShuPass） |
| 一言説明 | ES添削・志望動機・ガクチカ・企業別AI模擬面接をひとつにまとめた就活AIアプリ |
| URL | https://shupass.com |
| プラットフォーム | Web アプリ（レスポンシブ対応） |

### 1.2 ターゲットユーザー

- **年齢層**: 21〜23歳の日本の大学生（大学3年〜修士1年）
- **ペルソナ**: 就活塾には行かず、自力で進めたいが不安を抱える学生
- **具体的な悩み**:
  - ESの書き方が分からない
  - 志望動機を言語化できない
  - 複数社の締切を管理しきれない
  - 面接練習の相手がいない
  - AIで書くと「AIっぽさ」が残って不安

### 1.3 感情的ポジショニング

- **煽らない誠実さ**: 就活生の不安につけこまない、信頼できるツール
- **成果を約束しない**: 内定率・通過率は主張しない。機能の事実ベネフィットのみ
- **低ハードル**: カード登録不要、Free プランで試せる、失敗時は消費ゼロ

### 1.4 価格帯

| プラン | 月額 | 月次クレジット | 想定ユーザー |
|--------|------|---------------|-------------|
| Free | ¥0 | 50 | まず試したい方 |
| Standard | ¥1,490 | 350 | 本格的に就活対策したい方 |
| Pro | ¥2,980 | 750 | 複数企業を並行で対策したい方 |

**比較参考**: 就活塾は月3〜10万円（対面指導中心）

## 2. デザイン方向性

### 2.1 モデルサイト

**Stripe のプロダクトマーケティングページ**（stripe.com）をレイアウト・タイポ階層・境界線・複層シャドウ・セクションリズムの参照元とする。

### 2.2 キーワード

- 白基調、情報密度は高いが**余白で呼吸**
- 境界線とタイポグラフィで**金融グレードの信頼感**
- 煽りすぎない誠実さ（就活生向け）
- プロダクトスクリーンショットを文脈の中で見せる

### 2.3 採用しないもの

- Stripe のブランド紫（CTA 色は就活Pass 独自の赤）
- Stripe の Sohne フォント（Inter + Noto Sans JP を使用）
- 派手なアニメーション・パーティクル・3D

### 2.4 全体印象

**「慎重な学生が信頼できる、誠実な SaaS」**

### 6.8 FeatureInterviewSection（Feature 03: 対話 + 面接）

**役割**: 会話系 AI 機能の詳細。2パート構成
**背景**: `#f6f9fc`
**レイアウト**: 2パートに分割（パート間はボーダーで区切り）

**レイアウト**: 2カラム（左: テキスト / 右: スクリーンショット）

- Eyebrow: 志望動機・ガクチカ作成

- 本文: 志望動機やガクチカを、AIとの会話で言語化。頭の中の曖昧な考えが、ESに書ける材料に変わります。
- チェックリスト:
  1. 「なぜその業界か／なぜその会社か／そこで何をしたいか」など、志望動機に必要な観点を会話で順に整理
  2. 会社の事業内容や採用ページの情報を踏まえて、あなたらしい志望動機に近づける
  3. 「成長できる」「学べる」など、どの会社でも言えそうな表現は追加質問で具体化を促す

#### Part 2: 企業別 AI 模擬面接（ボーダーで区切り）

**レイアウト**: 2カラム（左: テキスト / 右: カード型 UI モック）

- チェックリスト:
  1. 会社の事業や採用ページの情報をふまえて質問を生成
  2. あなたの回答に応じて、深掘り質問 or 次の論点へ自動で切り替え
  3. 終了後に、良かった点・改善点・改善後の回答例・次に準備したい論点を提示
  4. 職種や面接方式（技術／ケース／人生史 など）、選考段階に合わせた質問

- 右: カード型の UI モック（スクリーンショットではなく、構造化されたカード）
