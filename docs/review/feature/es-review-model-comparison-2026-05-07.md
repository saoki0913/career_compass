### 6.10 横断的品質課題サマリ

#### 課題A: short band での系統的 shortfall（深刻度: Critical）

全テンプレートで short band（char_max ≤ 220）の出力が下限未達。原因の複合:

1. `_format_short_answer_guidance` の「短く終わらせない」指示が抽象的すぎる
2. Claude Sonnet の self-count が不正確で、内部目標帯の概念を無視して短く終了する
3. 最終文（展望/活かし方/接続）を書かずに終了する傾向が全テンプレートで一貫

`_length_control.py` の anthropic_claude profile で gap=8（short band）だが、実際の shortfall は12-22字。gap の設定ではなく、モデルが目標帯を無視する根本問題。

#### 課題B: 展望/応用/接続文の欠落（深刻度: High）

basic, gakuchika, self_pr, work_values で共通して SKELETON の最終セクション（展望/活かし方/学び）が欠落。モデルは「結論 + 根拠経験」の2文で満足してしまう。これが課題Aの文字数不足と直結している。

#### 課題C: AI臭フレーズの蔓延（深刻度: High）

STYLE_RULES §15「LLM特有フレーズは元回答にない限り使わない」の違反が横行。

| フレーズ | 出現回数/30 | 主な出現テンプレート |
|---|---|---|
| 「価値を/価値創出」 | 8 | company_motivation, basic, post_join_goals |
| 「社会課題に向き合い」 | 4 | company_motivation |
| 「前進させる/前に進める」 | 5 | gakuchika, self_pr |
| 「構造を整える」 | 3 | gakuchika, self_pr |
| 「認識をそろえる」 | 3 | self_pr |
| 「力を培った」 | 4 | gakuchika |
| 「貢献したい/する」 | 12+ | 全テンプレート |
| 「多様な関係者を巻き込みながら」 | 3 | company_motivation, self_pr |

現行の WATCH は禁止リストだが、代替表現の指示がないためモデルが別の AI フレーズに逃げる。

#### 課題D: company_motivation の企業固有性ゼロ（深刻度: High）

5ケース全てで企業固有の事業名・制度名・理念キーワードが0個。deep_grounding_requirements の「固有候補を最低1つ含める」指示がモデルに刺さっていない。RAG カードなしケースでの fallback guidance が「推測で企業固有情報を書かない」だが、代わりに何をすべきかが不明確。

#### 課題E: 成果の数値化不足（深刻度: Medium-High）

gakuchika の concrete_marker が参考ES平均 1.2-1.8個/文に対し 0-0.2個/文。成果を「意思決定までの時間を短縮」のように定性的に書き、数値（何%、何倍、何件）を伴わない。

#### 課題F: 構成の画一化（深刻度: Medium）

claude は「結論→経験→応用→貢献」の固定パターン。参考ESが示す多様な構成（二点構成型、経験架橋型、段階的キャリア像型、価値観宣言→複数実証型）を使い分けられていない。

### 6.12 モデル間比較の示唆

gemini-3.1-pro-preview が70% pass を達成できている要因の分析:

| 品質軸 | gemini の優位点 | claude-sonnet の課題 |
|---|---|---|
| 文字数制御 | 平均文字数204字でターゲット範囲内に収まる傾向 | 平均190字で一貫して下限を下回る |
| 構成遵守 | SKELETON の最終セクション（展望/活かし方）を省略しない | 2文で満足して最終セクションを欠落させる |
| 具体性 | 既存の文を説明密度を上げて拡張する | 既存の文を圧縮して短くまとめてしまう |
| AI臭さ | 同程度（「貴社」の使用、定型的な締め方は共通） | 同程度 |
| 処理時間 | 平均72秒（claude の4.5倍）— 時間をかけて文字数調整 | 平均16秒 — 高速だが文字数精度が犠牲 |

gemini が文字数 pass できているのは、処理時間の長さが示すように内部で複数回の self-revision を行っている可能性がある。一方で gemini も work_values_companyless_short（断片的エラー）や intern_reason_three_part_coverage_medium（234字/[210-220] 超過）で失敗しており、万能ではない。

gpt-5.4 は claude-sonnet と同じ 20% pass で、文字数不足パターンも酷似。gpt-5.4-mini は 6.7% で最も弱く、shortfall が全テンプレートで最大。

**結論**: 文字数制御の改善がpass率向上の最大レバーであり、プロンプト側の構造的改善（3文義務化、最終文強制、tiny delta 回復）で claude-sonnet のpass率を現行20%から50%以上に引き上げることが現実的な目標。品質面（AI臭さ、企業固有性、具体性）の改善は pass 後の次段階課題として位置づける。
