## 5. 根本原因分析

### 原因1: 文字数制御の構造的限界（最重要）

LLMは「150字以内」という上限制約は守れるが、「140〜150字」という下限＋上限の狭いバンドへの収束が困難。

| 制御段階 | 成功率 | 問題点 |
|---|---|---|
| 初回リライト | 13% (2/16) | ほぼ全件が下限を割る（LLMは短く書く傾向） |
| 3回リトライ後 | 31% (5/16) | length_focus_min プロンプトモードでも改善せず |
| length_fix後 | 50% (8/16) | ソフトリカバリ(90%フロア)で3件救済 |

GPT-5.4は8件中6件が文字数不足で失敗。400字目標で275字（69%）しか出力できないケースがある。Claude-sonnetは8件中2件の失敗に留まり、モデル間の文字数制御能力に大きな差がある。

### 原因2: 情報量ゼロの動詞チェーン

全出力で「整理した」「調整した」「見直した」「前進させた」が羅列されるが、何を・どう・いつが一切ない。

**出現頻度分析**:
| フレーズ | 出現率 |
|---|---|
| 「論点を整理」 | **50%** (8/16) |
| 「培った」 | **50%** (8/16) |
| 「認識をそろえ」 | **44%** (7/16) |
| 「構造化」 | **38%** (6/16) |
| 「関係者」 | **38%** (6/16) |
| 「価値創出」 | **31%** (5/16) |
| 「解像度」 | **25%** (4/16) |
| 「形にする」 | **25%** (4/16) |

「論点を整理」が全出力の半数に出現。ブロックリストでは解決できない（正当な日本語動詞）。問題は動詞自体ではなく、動詞の目的語や手段の具体性の欠如。

### 原因3: 構造的同型性（金太郎飴問題）

全出力が同一パターンに収束:
1. 抽象的な志望/取り組みの宣言
2. 「研究で〜」から始まる根拠
3. 「〜したい」「〜を担う」で終わる展望

リトライ回数が多い（平均3.5回）ほど「安全な」定型表現に収束し、悪化する。

**同一ユーザーの異なるテンプレートで冒頭文が完全一致**:
- Claude: ガクチカ(企業なし) と ガクチカ(企業あり) → 「研究室での進捗共有の仕組みを見直すことに最も力を入れた。」
- GPT: 同上 → 「研究室で進捗共有の型を見直し、情報の滞留を減らしたことである。」
- Claude版とGPT版の冒頭も同一（「事業と技術をつなぎ、構想を実装まで前に進める」）

### 原因4: 語彙レジスターの不整合

| 出力で使われる語彙 | 学生が実際に使う語彙 |
|---|---|
| 解像度、価値創出、推進力 | やりたい、面白い、自分に合ってる |
| 合意形成、構造化、判断軸 | みんなで話し合って、わかりやすく、自分なりに |
| 確信している | 思っている、感じた |

ESは学生の等身大の声を聞くためのもの。MBA的語彙は「本人が書いていない」シグナル。

### 原因5: 素材不足時の創作

元入力「幅広い事業に関わり、自分の視野を広げたい。」(20字) に対し、Claudeが「複数業界のインターンで事業構造の差異を体感した」と**元回答に存在しない経験を創作**。タイトバンドでは下限を満たすために字数を増やす圧力が強まり、素材不足の入力でハルシネーションリスクが上昇する。

### 検出漏れの主要ギャップ

| ギャップ | 内容 | 影響 |
|---|---|---|
| コンサル語彙コロケーション | 「論点を整理」「認識をそろえ」「構造化」「解像度」が未登録 | 最頻出AI指紋が素通り |

| 文末追跡の不足 | 5形式のみ追跡。「を担う」「を志望する」「させたい」等が未追跡 | 反復的な締めを検出できない |

| 新規パターン | ペナルティ | トリガー |
|---|---|---|
| `ai_formulaic_collocation` | 2.0 | 11フレーズ中2つ以上がユーザー原文に不在 |
| `verbal_tic` | 1.5 | 同一内容語が400字以下で4回以上出現 |
| `cross_model_template` | 2.5 | 10文字以上の逐語フレーズがテンプレートリストに一致 |

### 根本的な限界

ブロックリスト拡大には構造的限界がある。AI臭の本質は:
1. **情報密度の低さ**: 動詞の目的語・手段が欠如
2. **構文の均質性**: 全出力が同じ骨格
3. **レジスターの不整合**: 学生の声ではなくビジネス文書の声

## 7. 改善優先順位

| 優先度 | 改善項目 | 対象次元 | 現状→目標 | インパクト |
|--------|---------|---------|-----------|----------|
| **P0** | **文字数制御の改善** | D3 | 2.0→4.0 | **最大。パス率50%→90%** |
| **P1** | 具体性の質的改善 | D4 | 2.0→3.5 | 全出力の品質底上げ |
| **P2** | AI臭の根本対策 | D7 | 1.5→3.0 | リクルーターのAI判定回避 |
| **P3** | 差別化 + 金太郎飴防止 | D8 | 1.3→2.5 | 「この学生に会いたい」の創出 |
| **P4** | 業界別コーチング (10+2軸) | D5, D6 | +0.5 | 業界適合性向上 |
| **P5** | 面接深掘り耐性 | D9 | 1.3→2.5 | ES→面接の接続改善 |

### P0: 文字数制御の改善

パス率50%は提出不可レベル。

**対象ファイル**: `backend/app/prompts/es_templates.py`（L177-211 `_MODEL_FAMILY_DEFAULTS`）

1. **GPT-5.4 の内部目標ギャップ拡大**: `openai_gpt5` の short gap を 4→6、long gap を 9→15+ に拡大
2. **early recovery トリガー前倒し**: `early_length_fix_after_attempt` を GPT 向けに 3→2 に短縮
3. **length_focus_min の具体化**: 「目標字数に近づけて」→「あと20字追加して」形式に変更
4. **GPT long帯の特別対応**: 400字目標で275字しか出せない問題への max_tokens 調整
5. **self_pr medium帯の構造ガイド**: 両モデルが157/159字に収束する問題。4文構成ガイド追加
6. **soft recovery帯の拡大検討**: FINAL_SOFT_MIN_FLOOR_RATIO を 0.9→0.85。ただし品質妥協

### P1: 具体性の質的改善

- プロンプトに「映像化テスト」指示: 「読み手がその場面を映像として思い浮かべられるか確認。『整理した』ではなく『ホワイトボードに書き出して優先順を付けた』のように」
- `_count_concrete_markers` を多層化（数値だけでなく動作の目的語・手段・固有名詞も計測）
- `example_good` 自体の具体性引き上げ（現行example_goodがAI臭い）
- 否定指示→肯定指示への転換

### P2: AI臭の根本対策

**対象ファイル**: `backend/app/prompts/es_templates.py`（L40-91）、`backend/app/routers/es_review_validation.py`（L783-916）

- `_STYLE_RULES` に LLM禁止語追加: 「認識をそろえ」→「目線を合わせ」、「構造化」→禁止、「価値創出」→「成果を出す」、「解像度」→禁止
- AI臭検出に `ai_formulaic_collocation` + `verbal_tic` パターン追加
- レジスター制御: 「大学3-4年生の文体維持。MBA語彙禁止」
- 出現頻度制御: 「培った」→「身につけた/鍛えた/磨いた」に分散

### P3: 差別化 + 金太郎飴防止

- テンプレート定義に `differentiation_guidance` 追加
- **テンプレート間冒頭文分散**: 同一ユーザーのテンプレート間で冒頭一致を防止
- `anti_patterns` に「元回答の表現を2文以上で繰り返さない」ルール追加
- 文末表現多様性: 「〜と考える」「確信している」「実感した」を締め文での使用禁止

### P4: 業界別コーチング (10+2軸)

- 10コーチングカテゴリ + 日系/外資軸 + 企業規模軸
- RAG企業情報を優先し、業界ルールは補完的に適用
- 商社固有: 「当事者意識」「巻き込み力」「泥臭さ」の指示

### P5: 面接深掘り耐性

- gakuchika, self_pr テンプレートに `<interview_resilience>` ブロック追加
- 「expandable」な記述（面接で展開できるフック）の指示
