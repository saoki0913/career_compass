# `company_motivation`：企業志望理由の構成ヒント

このファイルは、品質フィルタ後の参考ESから抽出した構成パターンをもとに、文字数制限が変わっても高品質なESを書けるように整理したものです。

使い方はシンプルです。まず設問タイプを確認し、次に指定文字数帯の型を選び、最後に自分の経験・企業固有要素・成果数字を差し込んでください。

文字数帯は次の6段階に対応しています。

- `〜100字`
- `100〜200字`
- `200〜300字`
- `300〜400字`
- `400〜500字`
- `500字以上`


## 1. この設問で評価されること

`company_motivation` では、単なる企業理解ではなく、「自分が実現したいこと」と「その企業でなければならない理由」が一本の線でつながっているかが見られます。企業の理念や事業を褒めるだけでは弱く、自分の経験・問題意識・強みが、その企業の事業や環境でどう活きるのかまで示す必要があります。

評価されやすい文章は、次の問いに答えています。

- なぜその企業なのか
- その企業のどの特徴に惹かれているのか
- 自分の経験や関心とどう接続しているのか
- 入社後、どの領域でどう貢献したいのか

## 2. 基本の論理構成

```text
① 結論：貴社で〇〇を実現したい
② 背景：その思いを持った経験・問題意識
③ 企業固有性：事業、顧客基盤、技術、文化、実証環境など
④ 自己接続：自分の経験・強みがどう活きるか
⑤ 貢献：入社後にどの領域で価値を出すか
```

最も強い型は、次の流れです。

```text
実現したいこと
→ そう考えるようになった経験
→ その企業だから実現できる理由
→ 自分の強みとの接続
→ 入社後の貢献
```

## 3. 文字数帯別の設計

### 3-1. `〜100字`

目的は「志望理由の芯だけを示す」ことです。背景説明は最小限にし、結論・企業固有性・貢献を一文または二文でまとめます。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 結論 | 30% | 貴社で実現したいこと |
| 企業固有性 | 40% | その企業の一番重要な特徴 |
| 貢献 | 30% | 自分がどう価値を出すか |

**テンプレート**

```text
私は〇〇を実現したく、貴社を志望する。貴社の□□という強みを活かし、△△の経験で培った力を通じて、◇◇に貢献したい。
```

**削るもの**

- 詳細な原体験
- 企業説明の羅列
- 複数の強み

### 3-2. `100〜200字`

一つの原体験と一つの企業固有性を入れられます。企業の特徴を説明しすぎず、「自分の経験からなぜ惹かれたか」を優先します。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 結論 | 20% | 志望理由を明示 |
| 背景 | 30% | 関心が生まれた経験 |
| 企業固有性 | 30% | 事業・技術・顧客基盤など |
| 貢献 | 20% | どう活躍したいか |

**テンプレート**

```text
私は〇〇を実現したく、貴社を志望する。△△の経験を通じて、□□の重要性を実感した。貴社はAに強みを持ち、□□を社会に実装できる環境がある。私の◇◇力を活かし、〇〇の実現に貢献したい。
```

### 3-3. `200〜300字`

志望理由として最も扱いやすい文字数帯です。原体験、企業固有性、自己接続、貢献を一通り入れます。ただし、企業固有性は一つに絞ると密度が上がります。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 結論 | 15% | 実現したいこと |
| 背景 | 25% | 問題意識が生まれた経験 |
| 企業固有性 | 30% | なぜその企業か |
| 自己接続 | 15% | 自分の強み・経験 |
| 貢献 | 15% | 入社後の価値発揮 |

**テンプレート**

```text
私が貴社を志望する理由は、〇〇を実現したいからである。私は△△の経験を通じて、□□という課題を実感した。この課題を解決するには、AだけでなくBが必要だと考えている。貴社はCに強みを持ち、□□を実装できる環境がある。私は◇◇の経験で培った力を活かし、貴社で〇〇の実現に貢献したい。
```

### 3-4. `300〜400字`

「なぜ他社ではなくその企業か」を厚くできます。企業固有性を二つまで入れ、自己経験との接続を明確にします。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 結論 | 10% | 志望理由 |
| 背景 | 25% | 原体験・問題意識 |
| 企業固有性 | 35% | 事業・技術・文化など2点まで |
| 自己接続 | 15% | 自分の経験・強み |
| 貢献 | 15% | 具体領域での貢献 |

**テンプレート**

```text
私が貴社を志望する理由は、〇〇を実現したいからである。私は△△に取り組む中で、□□という課題に直面し、Aの重要性を実感した。貴社はBという事業基盤に加え、Cのような取り組みを通じて、□□を実社会に届けられる点に強みがあると考える。これは、私が目指す〇〇の実現に最も近い環境である。私はこれまで◇◇の経験で、課題を構造化し周囲を巻き込む力を培った。この力を活かし、貴社で△△領域の価値創出に貢献したい。
```

### 3-5. `400〜500字`

企業理解の深さと、自分の将来像を両方示せます。業界課題、企業固有性、自分の経験をつなげると説得力が出ます。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 結論 | 10% | 何を実現したいか |
| 問題意識 | 20% | 社会・業界・顧客課題 |
| 原体験 | 15% | なぜ自分が関心を持つのか |
| 企業固有性 | 30% | 他社との差別化要素 |
| 自己接続 | 15% | 経験・強み |
| 貢献 | 10% | 入社後の方向性 |

**テンプレート**

```text
私が貴社を志望する理由は、〇〇を通じて□□を実現したいからである。私は△△の経験を通じて、□□にはAという課題があると感じた。特に、Bの現場では、技術や仕組みがあってもCが不足することで価値が十分に届かない場面があると考えている。貴社はDという顧客基盤とEという技術・事業資産を持ち、課題発見から実装まで一貫して取り組める点に強みがある。この環境であれば、私が目指す〇〇を実現できると考えた。私は◇◇の経験で、複雑な課題を整理し、関係者を巻き込みながら成果を出す力を培った。入社後はこの力を活かし、△△領域で□□の価値創出に貢献したい。
```

### 3-6. `500字以上`

500字以上では、単に長くするのではなく「比較」「時間軸」「貢献領域」を追加します。企業説明が長くなりすぎると弱くなるため、各段落が志望理由を補強しているか確認してください。

**推奨構成**

```text
第1段落：結論と実現したいこと
第2段落：原体験・問題意識
第3段落：業界課題または社会課題の理解
第4段落：企業固有性。他社ではなく貴社である理由
第5段落：自分の強み・経験との接続
第6段落：入社後の貢献と中長期の方向性
```

**テンプレート**

```text
私が貴社を志望する理由は、〇〇を通じて□□を実現したいからである。

この思いは、△△の経験から生まれた。当時、私はAという課題に直面し、□□を実現するにはBだけでなくCが必要だと学んだ。

現在、□□の領域ではDという課題があり、単なるサービス提供ではなく、顧客や社会の状況に合わせて価値を実装する力が求められていると考える。

その中で貴社は、Eという事業基盤、Fという技術・人材、Gという顧客接点を持ち、課題発見から実装・改善まで一貫して取り組める点に強みがある。これは私が目指す〇〇の実現に最も適した環境である。

私はこれまで◇◇の経験を通じて、課題を構造化し、周囲を巻き込みながら成果を出す力を培ってきた。特に、HにおいてIを行い、Jという成果を得た経験は、貴社での業務にも活かせると考えている。

入社後は、まず△△領域で専門性と現場理解を磨き、将来的には□□の変革を主導できる人材として、貴社の価値創出に貢献したい。
```

## 4. 文字数を増やすときに足す順番

1. 企業固有性を具体化する
2. 原体験を一文追加する
3. 業界・顧客課題を追加する
4. 自分の強みの根拠を追加する
5. 入社後の貢献領域を具体化する

## 5. 文字数を削るときに残す順番

1. 実現したいこと
2. 企業固有性
3. 自分の経験・強み
4. 貢献方向
5. 原体験の詳細

## 6. NGパターン

```text
貴社の理念に共感した。貴社は社会に大きな影響を与えている。私は成長したい。貴社で貢献したい。
```

弱い理由は、企業名を変えても成立し、自分の経験と企業固有性が接続していないためです。

## 7. 最終チェック

- 企業名を他社に置き換えても成立しない内容になっているか
- 企業説明より、自分との接続に字数を使っているか
- 原体験が志望理由の根拠になっているか
- 入社後にどの領域で価値を出すかが明確か
- 「共感した」「魅力を感じた」だけで終わっていないか
