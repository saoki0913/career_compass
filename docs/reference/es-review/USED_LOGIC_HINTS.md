## `basic`：汎用（型に当てはまらないES）

### この設問で意識する品質（quality_hints・型レベル）

- 冒頭1文で設問への答え（結論）を言い切る（前置きや設問の言い換えから始めない）
- 結論の後に、根拠となる経験・行動を具体的に示す
- 抽象語だけで終えず、読み手がイメージできる材料（数値・場面・行動）を入れる
- 1文に複数論点を詰め込まず、各文に役割を持たせる
- 価値観や主張は、行動例や結果とセットで示す
- 設問が問うていることに最後まで答え続ける（脱線しない）
- 最後に仕事や今後への接続を短く置く
- 文末は3文ごとに語尾を変える（〜だ／〜である／体言止め／〜した を交互に使う）

### 文レベルの流れ（sentence_flow・型レベル）

- 設問への答えを結論として言い切る
- その結論を裏づける経験・行動を具体的に述べる
- 行動の結果や学びを示す
- 仕事や今後への接続・応用で締める
- 接続: 結論→根拠→結果→接続を一本の線でつなぎ、前文の要素を次文の起点にする

### 論理構成パターン（logic_patterns・型レベル・件数表示なし）

- 主な論理アプローチ: **結論先行・具体根拠型**
  - 設問への答えを冒頭で言い切り、根拠となる経験・行動→結果・学び→仕事や今後への接続、を一本の線でつなぐ汎用構成
  - 構成設計: 結論→背景→経験・行動（具体）→結果・学び→今後への接続
  - 根拠提示: 主張は行動例や結果とセットで示し、抽象語だけで終えない
  - 接続パターン: 結論→根拠→結果→接続を因果でつなぎ、設問に最後まで答え続ける
- 説得力の鍵: 結論が冒頭にあり、抽象語でなく具体（数値・場面・行動）で裏づけられていること
- 配分傾向: 結論15% / 背景20% / 根拠45% / 接続20%（300〜400字基準）
- 冒頭の型: 設問への答えを一文で言い切る
- 締めの型: 仕事や今後への接続で短く締める
- 品質指標（最終チェック由来）:
  - 設問への答えが冒頭で明確である
  - 根拠が具体的な経験・行動で示されている
  - 抽象語だけで終わっていない
  - 各文に役割があり、1文1論点になっている
  - 仕事や今後への接続がある
- よくある弱点（NG由来）:
  - 設問に正面から答えず、前置きや一般論が長い
  - 抽象語の羅列で、読み手が具体をイメージできない
  - 1文に複数論点を詰め込み、主張の芯がぼやけている

## `intern_goals`：インターンで学びたいこと

### この設問で意識する品質（quality_hints・型レベル）

- 冒頭1文でインターンで学びたいこと・達成したいことを言い切る（抽象的な導入や設問の言い換えから始めない）
- 学びたい対象は一つか二つに絞る（「多くを学びたい」で終えない）
- 学びたい内容を業務・工程・技術・視点のいずれかに分解して具体化する
- なぜそれを学びたいのかを過去経験に基づいて示す
- インターンでしか得られない学びであることを示す
- インターン名は冒頭1回までにし、以降は「本インターンシップ」「本プログラム」に言い換える
- 学んだことを将来どう活かすかまでつなぐ
- 文末は3文ごとに語尾を変える（〜だ／〜である／体言止め／〜した を交互に使う）

### 文レベルの流れ（sentence_flow・型レベル）

- インターンで学びたいこと・達成したいことを言い切る（結論）
- そう考えるようになった過去経験を述べる
- 現在の自分に足りないものを示す
- 本プログラムで具体的に得たい学び（業務・工程・技術・視点）を示す
- その学びを将来どう活かすかで締める
- 接続: 学びたいこと→経験→不足→具体的な学び→将来活用を因果でつなぎ、学びの対象を絞る

### 論理構成パターン（logic_patterns・型レベル・件数表示なし）

- 主な論理アプローチ: **学習目標→経験→具体的学び型**
  - 学びたいことを冒頭で言い切り、そう考えた経験→現在の不足→本プログラムで具体的に得たい学び→将来への活用、をつなぐ構成
  - 構成設計: 学習目標→そう考えた経験→現在の不足→具体的な学び（分解）→将来活用
  - 根拠提示: 学びたい内容を抽象語で終えず業務・工程・技術・視点に分解し、過去経験を目標の根拠とする
  - 接続パターン: 学びたいこと→経験→不足→具体的な学び→将来活用を因果でつなぐ
- 説得力の鍵: 学びたい対象が一つか二つに絞られ、業務・工程・技術・視点に分解されていること
- 配分傾向: 目標15% / 目標1 25% / 目標2 25% / 背景20% / 活用15%（300〜400字・目標二つの場合）
- 冒頭の型: インターンで学びたいことを一文で言い切る
- 締めの型: 学びを将来どう活かすかで締める
- 品質指標（最終チェック由来）:
  - 冒頭で「何を学びたいか」が明確である
  - 学びたい内容が業務・工程・技術・視点に分解されている
  - 過去経験と学習目標がつながっている
  - インターンでしか得られない学びがある
  - 将来像への接続がある
- よくある弱点（NG由来）:
  - 学びたい対象が具体化されておらず、参加後に何を得たいのか不明確
  - 「多くのことを学びたい」「刺激を受けたい」等の抽象表現で終わっている
  - 学びたいことの羅列で、対象が絞られていない

### 文字数帯別の骨子（skeleton・char_max で選択）

**〜100字**

- 目標（約40%）: 学びたいこと
- 必要性（約30%）: なぜ必要か
- 活用（約30%）: 将来どう使うか

**100〜200字**

- 目標（約25%）: 何を学びたいか
- 背景（約25%）: なぜその目標か
- 具体対象（約30%）: 業務・工程・技術
- 活用（約20%）: 将来への接続

**200〜300字**

- 目標（約20%）: 学習目標
- 背景（約25%）: 過去経験・問題意識
- 具体対象（約30%）: 体験したい業務・工程
- 自己課題（約15%）: 今の不足
- 活用（約10%）: 将来像

**300〜400字（既定）**

- 目標（約15%）: 全体目標
- 目標1（約25%）: 技術・工程理解
- 目標2（約25%）: 顧客・ビジネス視点
- 背景（約20%）: その目標を持つ理由
- 活用（約15%）: 将来への接続

**400〜500字**

- 目標（約10%）: 達成したいこと
- 原体験（約20%）: なぜ学びたいか
- 現在の課題（約20%）: 自分に不足する視点
- 具体的な学び（約30%）: 業務・工程・技術・社員との協働
- 将来活用（約20%）: キャリア・貢献

**500字以上**

- 第1段落: 達成したい目標の結論
- 第2段落: 目標を持った背景
- 第3段落: 現在の自分の強みと不足
- 第4段落: インターンで学びたい具体内容
- 第5段落: 参加中の行動方針
- 第6段落: 将来への活用

## `work_values`：働くうえで大切にしたい価値観

### この設問で意識する品質（quality_hints・型レベル）

- 冒頭1文で大切にしている価値観を一言で言い切る（設問の言い換えや一般論から始めない）
- 価値観は抽象語のままにせず、行動で示せる言葉に具体化する
- その価値観が形成された実体験を一つ示す
- 経験から得た学びを、仕事観として一段抽象化する
- 経験の説明を長くしすぎず、価値観の説明に従属させる
- 志望企業・職種でその価値観がなぜ重要かにつなぐ
- 「大切にしたい」で終えず、どう行動するかまで書く
- 文末は3文ごとに語尾を変える（〜だ／〜である／体言止め／〜した を交互に使う）

### 文レベルの流れ（sentence_flow・型レベル）

- 大切にしている価値観を一言で言い切る（結論）
- その価値観が形成された実体験を述べる
- 経験から得た学びを仕事観として抽象化する
- 志望企業・職種でその価値観がなぜ重要かにつなぐ
- 入社後にどう行動して体現するかで締める
- 接続: 価値観→形成経験→学び→仕事での体現を因果でつなぎ、経験の説明は価値観に従属させる

### 論理構成パターン（logic_patterns・型レベル・件数表示なし）

- 主な論理アプローチ: **価値観→実体験→仕事観・体現型**
  - 大切にする価値観を冒頭で一言で定義し、形成された実体験→得た学びの抽象化→企業・職種での重要性→入社後の体現、をつなぐ構成
  - 構成設計: 価値観の定義→形成された原体験→仕事観への抽象化→企業・職種での重要性→体現方法
  - 根拠提示: 価値観は行動で示せる言葉に具体化し、実体験を一つ示す。経験の説明は価値観の説明に従属させる
  - 接続パターン: 価値観→経験→学び→仕事での体現を因果でつなぎ、各文で価値観に戻る
- 説得力の鍵: 価値観が抽象語でなく実体験に裏づけられ、仕事での具体行動まで落ちていること
- 配分傾向: 価値観10% / 原体験30% / 学び25% / 企業接続20% / 体現15%（300〜400字基準）
- 冒頭の型: 大切にする価値観を一言で言い切る
- 締めの型: 入社後にどう行動して価値観を体現するかで締める
- 品質指標（最終チェック由来）:
  - 価値観が一言で明確である
  - 価値観が実体験に基づいている
  - 経験の説明が長すぎず、価値観の説明に従属している
  - 企業や職種でその価値観がどう活きるか書かれている
  - 「大切にしたい」だけでなく、どう行動するかが明確である
- よくある弱点（NG由来）:
  - 価値観の根拠となる経験も、仕事での行動イメージもない
  - 経験の説明が長すぎてガクチカのようになり、価値観の説明に戻れていない
  - 「挑戦」「成長」等の抽象語の羅列で終わっている

### 文字数帯別の骨子（skeleton・char_max で選択）

**〜100字**

- 価値観（約35%）: 何を大切にするか
- 根拠（約30%）: 経験から得た学び
- 体現（約35%）: 仕事でどう行動するか

**100〜200字**

- 価値観（約20%）: 一言で明示
- 経験（約30%）: 形成された背景
- 学び（約25%）: 経験から得た仕事観
- 体現（約25%）: 志望先での行動

**200〜300字**

- 価値観（約15%）: 大切にする姿勢
- 経験（約30%）: 価値観が形成された場面
- 学び（約25%）: 仕事観への抽象化
- 企業接続（約20%）: その価値観が活きる業務
- 貢献（約10%）: どう体現するか

**300〜400字（既定）**

- 価値観（約10%）: 一言で示す
- 原体験（約30%）: 価値観が形成された経験
- 学び（約25%）: 抽象化された仕事観
- 企業接続（約20%）: その価値観が重要な理由
- 体現（約15%）: 入社後の行動

**400〜500字**

- 価値観（約10%）: 大切にしたいこと
- 原体験（約25%）: 形成された経験
- 行動・成果（約20%）: その価値観が表れた行動
- 学び（約20%）: 仕事観への抽象化
- 企業接続（約15%）: 企業・職種での重要性
- 体現（約10%）: どう行動するか

**500字以上**

- 第1段落: 大切にしたい価値観
- 第2段落: 価値観が形成された経験
- 第3段落: 経験中の行動と成果
- 第4段落: 経験から得た仕事観
- 第5段落: 企業・職種でその価値観が重要な理由
- 第6段落: 入社後の体現方法
