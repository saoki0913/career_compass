# `self_pr`：自己PR・強み／弱みの構成ヒント

このファイルは、品質フィルタ後の参考ESから抽出した構成パターンをもとに、文字数制限が変わっても高品質なESを書けるように整理したものです。

使い方はシンプルです。まず設問タイプを確認し、次に指定文字数帯の型を選び、最後に自分の経験・企業固有要素・成果数字を差し込んでください。

文字数帯は次の6段階に対応しています。

- `〜100字`
- `100〜200字`
- `200〜300字`
- `300〜400字`
- `400〜500字`
- `500字以上`


## 1. この設問で評価されること

`self_pr` では、自分の強みを一言で定義し、その強みが実際の経験で再現性を持って発揮されているかが見られます。`gakuchika` と異なり、主役は経験ではなく「強み」です。

評価されやすい回答は、次の条件を満たします。

- 強みが一言で明確に定義されている
- 強みが抽象的すぎない
- 具体経験で証明されている
- 成果が数字または状態変化で示されている
- 応募先の業務でどう活かすかまで書かれている

## 2. 基本の論理構成

```text
① 強みを一言で定義する
② 強みが発揮された経験を示す
③ 課題・行動・成果で証明する
④ 強みの再現性を示す
⑤ 応募先の業務でどう活かすかを書く
```

強い流れは次の通りです。

```text
強み
→ 強みが発揮された経験
→ 課題
→ 行動
→ 成果
→ 応募先での活かし方
```

## 3. 強みの表現を具体化する

| 弱い強み表現 | 強い強み表現 |
|---|---|
| コミュニケーション力 | 立場の異なる人の意見を整理し、合意形成に導く力 |
| リーダーシップ | 課題を構造化し、メンバーの協力を引き出して成果に導く力 |
| 課題解決力 | 複雑な課題を原因に分解し、再現性ある仕組みで解決する力 |
| 粘り強さ | 反対意見にも向き合い、データと対話で周囲を巻き込む力 |
| 成長意欲 | 未経験領域でも自ら学び、チームに還元して成果を出す力 |

## 4. 文字数帯別の設計

### 4-1. `〜100字`

強み、証明経験、活かし方を最短で書きます。複数の強みは出さず、一つに絞ります。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 強み | 30% | 一言で定義 |
| 証明 | 45% | 行動・成果 |
| 活用 | 25% | 応募先でどう活かすか |

**テンプレート**

```text
私の強みは〇〇力である。△△では、□□という課題に対しAを実行し、Bという成果を出した。この力を活かし、貴社でもCに貢献したい。
```

### 4-2. `100〜200字`

強みを証明する経験を一つ入れます。成果は一文で示し、最後に応募先での再現性を書きます。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 強み | 20% | 強みの定義 |
| 経験 | 25% | 強みが発揮された場面 |
| 行動 | 30% | 自分の具体行動 |
| 成果 | 15% | 結果 |
| 活用 | 10% | 応募先での活かし方 |

**テンプレート**

```text
私の強みは、〇〇力である。この強みは△△の経験で発揮された。当時、□□という課題があり、私はAとBを実行した。その結果、Cという成果を得た。この強みを活かし、貴社でもDの場面で周囲を巻き込み、価値創出に貢献したい。
```

### 4-3. `200〜300字`

強みの定義、課題、行動、成果、再現性を一通り入れられます。`gakuchika` になりすぎないよう、各文で強みに戻る意識が必要です。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 強み | 15% | 一言で定義 |
| 経験 | 20% | 発揮場面 |
| 課題 | 20% | 何が難しかったか |
| 行動 | 25% | 強みを活かした施策 |
| 成果 | 10% | 数字・状態変化 |
| 活用 | 10% | 業務での再現性 |

**テンプレート**

```text
私の強みは、〇〇力である。この強みは△△の経験で発揮された。当時、□□という課題があり、目標達成が難しい状況だった。私は原因をAと捉え、BとCを実行した。特に、Dを工夫することで周囲の協力を引き出した。その結果、Eという成果を得た。この経験から、私は複雑な課題でも自ら考え、周囲を巻き込みながら成果に結びつけられると考えている。貴社でもこの強みを活かし、Fに貢献したい。
```

### 4-4. `300〜400字`

強みの再現性を深く示せます。経験を一つに絞り、「なぜその行動が強みの証明になるのか」を明確にします。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 強み | 10% | 強みの定義 |
| 経験・課題 | 25% | 発揮場面と難しさ |
| 行動 | 30% | 強みを活かした行動 |
| 成果 | 15% | 結果 |
| 再現性 | 10% | どんな場面でも使える力 |
| 応募先活用 | 10% | 仕事での活かし方 |

**テンプレート**

```text
私の強みは、〇〇力である。この強みは△△の経験で発揮された。当時、□□という課題があり、チーム全体の成果に影響していた。私は、課題の原因がAとBにあると考えた。そこで、第一にCを行い、状況を可視化した。第二にDを実施し、関係者の協力を得ながら改善を進めた。特に、意見が分かれる場面では、Eを示して合意形成を図った。その結果、FがGに改善した。この経験から、私は課題を構造化し、周囲を巻き込みながら成果に導く力を持つと考えている。貴社でもこの強みを活かし、Hの業務で価値を発揮したい。
```

### 4-5. `400〜500字`

強みの定義、証明経験、別場面での再現性、応募先での活用まで書けます。長くなるほど強みがぼやけやすいため、強みは一つに固定します。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 強み | 10% | 一言で定義 |
| 経験 | 15% | 発揮場面 |
| 課題 | 15% | 難しさ |
| 行動 | 30% | 具体行動 |
| 成果 | 10% | 結果 |
| 再現性 | 10% | 他場面でも発揮できる理由 |
| 活用 | 10% | 応募先での活かし方 |

**テンプレート**

```text
私の強みは、〇〇力である。これは、△△の経験で発揮された。当時、□□という課題があり、Aの達成が難しい状況だった。私は、問題の原因をBとCに分解したうえで、第一にDを行い、課題を可視化した。第二にEを実施し、関係者の協力を得ながら改善を進めた。実行時には、Fという反対意見もあったが、目的と効果を丁寧に説明し、役割分担を見直すことで合意形成を行った。その結果、GがHに改善し、目標を達成した。この経験から、私は状況を客観的に捉え、周囲を巻き込みながら成果につなげる力を持つと考えている。この強みは、貴社のIの業務においても、関係者と協働しながら課題解決を進める場面で活かせると考えている。
```

### 4-6. `500字以上`

500字以上では、強みを一つに絞ったまま、複数の経験で再現性を示すこともできます。ただし、経験を二つ入れる場合は、どちらも同じ強みを証明する内容にしてください。

**推奨構成**

```text
第1段落：強みの定義
第2段落：強みが発揮された主経験
第3段落：課題と行動
第4段落：成果
第5段落：別場面での再現性、または強みの深掘り
第6段落：応募先での活かし方
```

**テンプレート**

```text
私の強みは、〇〇力である。これは、単にAができるということではなく、Bの状況でもCを行い、成果につなげる力である。

この強みは、△△の経験で発揮された。当時、□□という課題があり、目標であるDの達成が難しい状況だった。

私はまず、課題の原因をEとFに分解した。そのうえで、第一にGを行い、問題の構造を可視化した。第二にHを実行し、関係者が納得して行動できる状態を作った。実行時にはIという困難もあったが、Jを工夫することで協力を得た。

その結果、KがLに改善し、Mという成果を得た。

また、Nの場面でも同様に、状況を整理し、周囲の協力を引き出すことでOを達成した。このように、私の強みは一度限りの経験ではなく、異なる場面でも再現できる力である。

貴社では、Pの業務において多様な関係者と連携しながら課題解決を進める必要があると考えている。私はこの強みを活かし、相手の状況を理解しながら合意形成を進め、成果創出に貢献したい。
```

## 5. 弱みも聞かれた場合の型

```text
一方で、私の短所は〇〇である。過去には△△の場面で□□という課題があった。現在は、Aを意識し、Bの方法で改善している。この改善により、強みである〇〇をより効果的に発揮できるよう努めている。
```

弱みを書く場合は、弱みだけで終えず、改善行動まで必ず入れます。

## 6. 文字数を増やすときに足す順番

1. 強みの定義を具体化する
2. 強みが発揮された課題を説明する
3. 行動の理由を追加する
4. 成果を定量化する
5. 応募先での活かし方を具体化する

## 7. 文字数を削るときに残す順番

1. 強み
2. 証明経験
3. 自分の行動
4. 成果
5. 応募先での活用

## 8. NGパターン

```text
私の強みはコミュニケーション力とリーダーシップと粘り強さである。多くの経験でこれらを発揮してきた。
```

弱い理由は、強みが多すぎて焦点がぼやけ、どの強みを評価すべきか分からないためです。

## 9. 最終チェック

- 強みが一言で明確に定義されているか
- 強みが抽象的すぎないか
- 具体的な経験で証明されているか
- 成果が数字または状態変化で示されているか
- 応募先でどう活かすかまで書かれているか
- 強みを複数並べすぎていないか
