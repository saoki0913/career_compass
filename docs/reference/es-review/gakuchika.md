# `gakuchika`：学生時代に力を入れたこと・挑戦経験の構成ヒント

このファイルは、品質フィルタ後の参考ESから抽出した構成パターンをもとに、文字数制限が変わっても高品質なESを書けるように整理したものです。

使い方はシンプルです。まず設問タイプを確認し、次に指定文字数帯の型を選び、最後に自分の経験・企業固有要素・成果数字を差し込んでください。

文字数帯は次の6段階に対応しています。

- `〜100字`
- `100〜200字`
- `200〜300字`
- `300〜400字`
- `400〜500字`
- `500字以上`


## 1. この設問で評価されること

`gakuchika` では、活動の規模や華やかさよりも、困難に対してどのように考え、周囲を巻き込み、成果を出したかが見られます。高品質な回答には、課題の定量化・原因分析・具体施策・成果・学びが含まれています。

評価されるポイントは次の通りです。

- 課題が具体的である
- 原因分析がある
- 自分が主導した行動が明確である
- 周囲をどう巻き込んだかが見える
- 成果が数字または状態変化で示されている
- 学びが仕事でも再現できる強みに抽象化されている

## 2. 基本の論理構成

```text
① 経験の概要：何に取り組んだか
② 課題：どのような困難があったか
③ 原因分析：なぜその問題が起きていたか
④ 目標：何を達成しようとしたか
⑤ 行動：自分が主導した施策
⑥ 困難・反対意見：行動の中で何が難しかったか
⑦ 成果：数字または状態変化で示す
⑧ 学び：仕事でも再現できる強みとしてまとめる
```

強い流れは次の通りです。

```text
取り組み
→ 課題
→ 原因分析
→ 施策
→ 巻き込み
→ 成果
→ 学び
```

## 3. 強い要素の例

| 要素 | 使いやすい表現 |
|---|---|
| 定量課題 | 作業時間の約3割、受け取り率20%未満、期限内リリースが困難など |
| 原因分析 | 属人化、知識レベルの差、環境変化への対応遅れ、合意形成不足など |
| 施策分解 | 標準化、技術力向上、データ提示、個別交流、役割分担など |
| 巻き込み | 反対意見への説明、負担軽減、合意形成、協力者の獲得など |
| 成果 | 開発速度2倍、期限内リリース、受け取り率80%以上、入会者30名など |
| 学び | 構造化、粘り強い合意形成、組織を巻き込む力、仕組み化など |

## 4. 文字数帯別の設計

### 4-1. `〜100字`

経験の概要、行動、成果に絞ります。原因分析や学びは一語で入れる程度にします。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 概要 | 30% | 何に取り組んだか |
| 行動 | 40% | 自分が何をしたか |
| 成果・学び | 30% | 数字または得た力 |

**テンプレート**

```text
私は〇〇で△△に取り組んだ。□□という課題に対し、AとBを実行した結果、Cを達成した。この経験から、課題を構造化し成果につなげる力を培った。
```

### 4-2. `100〜200字`

課題と原因分析を一文ずつ入れられます。施策は二つまでに絞り、成果は必ず入れます。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 概要 | 15% | 取り組み |
| 課題 | 25% | 困難・目標未達 |
| 原因 | 20% | 問題の原因 |
| 行動 | 25% | 施策 |
| 成果・学び | 15% | 結果と再現性 |

**テンプレート**

```text
私は〇〇に取り組んだ。当初、△△により□□が困難な状況だった。原因をAと捉え、①〇〇、②〇〇を実行した。特に、反対するメンバーにはBを示して協力を得た。その結果、CからDへ改善し、□□を達成した。この経験から、〇〇力を培った。
```

### 4-3. `200〜300字`

`gakuchika` として最低限の完成度を出しやすい文字数帯です。課題、原因、行動、成果、学びを一通り入れます。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 概要 | 10% | 何に取り組んだか |
| 課題 | 20% | 難しさ |
| 原因分析 | 20% | なぜ起きたか |
| 行動 | 25% | 自分の施策 |
| 成果 | 15% | 定量・状態変化 |
| 学び | 10% | 再現性 |

**テンプレート**

```text
私は〇〇で△△に取り組んだ。当初、Aという状況により、□□の達成が難しかった。私は原因をBとCに分解し、二つの施策を実行した。第一に、〇〇を行い、△△を改善した。第二に、□□を行い、チーム全体の行動を変えた。その結果、DがEに改善し、目標を達成した。この経験から、課題を構造化し、周囲を巻き込みながら成果を出す力を培った。
```

### 4-4. `300〜400字`

困難や反対意見を入れられます。単なる成功談ではなく、葛藤を乗り越えた過程を書くと評価されやすいです。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 概要 | 10% | 取り組み |
| 課題・目標 | 20% | 何が難しかったか |
| 原因分析 | 20% | 自分の見立て |
| 行動 | 25% | 施策2つ程度 |
| 巻き込み | 10% | 反対・困難への対応 |
| 成果・学び | 15% | 結果と再現性 |

**テンプレート**

```text
私は〇〇において、△△という課題の解決に取り組んだ。当初、Aという状況により、□□の達成が困難だった。私は原因をBとCに分解した。そこで、第一に〇〇を行い、情報や手順を標準化した。第二に□□を実施し、メンバーが自発的に行動できる仕組みを整えた。実行時には、Eという反対意見もあったが、Fを示し、個別に対話することで協力を得た。その結果、GがHに改善し、目標を達成した。この経験から、課題を分析し、周囲を巻き込みながら成果を出す力を培った。
```

### 4-5. `400〜500字`

背景、課題、原因、施策、困難、成果、学びを十分に書けます。特に「自分がなぜその行動を選んだか」を入れると、思考力が伝わります。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 背景・概要 | 10% | 活動の前提 |
| 課題・目標 | 20% | 何を達成したかったか |
| 原因分析 | 20% | 問題の構造 |
| 行動 | 25% | 施策と自分の役割 |
| 困難対応 | 10% | 巻き込み・合意形成 |
| 成果 | 10% | 定量成果 |
| 学び | 5% | 再現性 |

**テンプレート**

```text
私は〇〇で△△に取り組んだ。活動では□□を目標としていたが、当初はAという状況により達成が難しかった。私は、問題の原因がBとCにあると考えた。そこで、第一に〇〇を行い、作業や判断の基準を明確にした。第二に□□を実施し、メンバー間の認識差を減らした。これらの施策を選んだのは、単発の努力ではなく、誰でも継続できる仕組みに変える必要があると考えたためである。実行時には、負担増を懸念する声もあったが、目的と期待効果を示し、役割分担を調整することで協力を得た。その結果、DがEに改善し、目標を達成した。この経験から、課題を構造化し、周囲を巻き込みながら成果を出す力を培った。
```

### 4-6. `500字以上`

500字以上では、単なる時系列ではなく「思考の深さ」を示します。背景説明が長くなりすぎないよう、課題・原因・行動に字数を使います。

**推奨構成**

```text
第1段落：取り組みの概要と役割
第2段落：課題と目標
第3段落：原因分析
第4段落：施策1
第5段落：施策2と巻き込み
第6段落：成果
第7段落：学びと再現性
```

**テンプレート**

```text
私が学生時代に力を入れたことは、〇〇における△△の改善である。私は□□という立場で、Aを目標に活動した。

当初、Bという課題があり、目標達成が難しい状況だった。具体的には、Cが原因でDが発生し、チーム全体の成果に影響していた。

私はこの問題の原因を、EとFの二点に分解した。Eについては、判断基準が共有されておらず、個人の経験に依存していた。Fについては、メンバー間の目的意識に差があり、行動量にばらつきがあった。

そこで第一に、〇〇を実行した。これにより、誰が見ても同じ基準で判断できるようにし、作業の属人化を減らした。

第二に、□□を実行した。実行時には、負担が増えることへの反対もあったが、Gを示し、個別に対話することで協力を得た。また、メンバーの負担を抑えるためにHを工夫した。

その結果、IがJに改善し、Kという成果を達成した。数字で示せない場合でも、Lという状態変化が生まれた。

この経験から、私は課題を構造化し、関係者を巻き込みながら再現性ある仕組みを作る力を培った。この力は、仕事においても複雑な課題に向き合い、成果につなげる場面で活かせると考えている。
```

## 5. 文字数を増やすときに足す順番

1. 課題の具体性・定量化
2. 原因分析
3. 施策を選んだ理由
4. 巻き込み・合意形成の過程
5. 成果の定量化・学びの再現性

## 6. 文字数を削るときに残す順番

1. 取り組みの概要
2. 課題
3. 自分の行動
4. 成果
5. 学び
6. 背景説明

## 7. NGパターン

```text
私はサークル活動を頑張った。メンバーと協力して新歓を行い、多くの新入生を集めることができた。この経験から協調性を学んだ。
```

弱い理由は、課題・原因・自分の具体行動・成果が見えず、本人の思考プロセスが伝わらないためです。

## 8. 最終チェック

- 課題が具体的に書かれているか
- 課題の原因を分析しているか
- 自分が主導した行動が明確か
- 施策が2〜3個に整理されているか
- 困難や反対意見をどう乗り越えたかがあるか
- 成果が数字または状態変化で示されているか
- 最後に、仕事でも再現できる強みとして抽象化されているか
