# `role_course_reason`：職種・コース・テーマ志望理由の構成ヒント

このファイルは、品質フィルタ後の参考ESから抽出した構成パターンをもとに、文字数制限が変わっても高品質なESを書けるように整理したものです。

使い方はシンプルです。まず設問タイプを確認し、次に指定文字数帯の型を選び、最後に自分の経験・企業固有要素・成果数字を差し込んでください。

文字数帯は次の6段階に対応しています。

- `〜100字`
- `100〜200字`
- `200〜300字`
- `300〜400字`
- `400〜500字`
- `500字以上`


## 1. この設問で評価されること

`role_course_reason` では、企業への志望理由ではなく、「なぜその職種・コース・テーマを選ぶのか」が問われます。職種理解と自己適性の接続が重要です。

評価されやすい回答は、次の3点を満たしています。

- その職種・コースの役割を理解している
- 自分の経験・強み・関心と接続している
- その職種で実現したい価値が明確である

## 2. 基本の論理構成

```text
① 結論：その職種・コースを志望する理由
② 役割理解：その職種が何を担うのか
③ 自己接続：経験・強み・関心との接点
④ 具体業務：そのコースで取り組みたいこと
⑤ 将来像：その職種でどう価値を出すか
```

強い流れは次の通りです。

```text
職種を選ぶ理由
→ 職種の役割理解
→ 自分の経験・適性
→ 取り組みたい業務
→ 将来の価値発揮
```

## 3. 職種別に入れると強い観点

| 職種・コース系統 | 入れると強い観点 |
|---|---|
| エンジニア | 技術領域、開発工程、設計思想、運用視点、ユーザー価値 |
| IT・デジタル企画 | 業務改善、顧客体験、データ活用、事業部門との橋渡し |
| コンサルタント | 課題特定、仮説構築、提案、実行支援、クライアント変革 |
| マーケティング | 顧客理解、データ分析、価値提案、ブランド・事業成長 |
| 金融・商社系専門職 | 産業理解、リスク・収益、社会インフラ性、顧客基盤 |

## 4. 文字数帯別の設計

### 4-1. `〜100字`

職種選択の理由と自己適性を一文で示します。企業志望理由に寄らないよう、職種名・業務内容を必ず入れます。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 結論 | 35% | なぜその職種か |
| 自己適性 | 35% | 自分の経験・強み |
| 価値発揮 | 30% | どう貢献するか |

**テンプレート**

```text
〇〇職を志望する理由は、□□を実現したいからだ。△△の経験で培った◇◇力を活かし、Aの業務を通じて顧客・社会に価値を届けたい。
```

### 4-2. `100〜200字`

職種の役割理解を一文入れます。「成長できそう」ではなく、職種で担う価値に惹かれていることを示します。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 結論 | 20% | 志望理由 |
| 役割理解 | 30% | その職種が担う価値 |
| 自己接続 | 30% | 経験・強みとの接続 |
| 将来像 | 20% | どう活躍したいか |

**テンプレート**

```text
〇〇職を志望する理由は、□□を実現したいからだ。この職種は、Aを通じて顧客課題を解決する役割だと理解している。私は△△の経験で◇◇力を培い、課題を形ある成果に変えることにやりがいを感じた。この力を活かし、貴社で□□に貢献したい。
```

### 4-3. `200〜300字`

役割理解、自己適性、取り組みたい業務を一通り入れます。職種の説明と自分の経験が分離しないように注意します。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 結論 | 15% | 職種志望理由 |
| 役割理解 | 25% | 職種の価値 |
| 経験 | 25% | 自分の適性の根拠 |
| 具体業務 | 20% | 取り組みたいこと |
| 将来像 | 15% | 価値発揮 |

**テンプレート**

```text
私が〇〇職を志望する理由は、□□を実現したいからである。〇〇職は、AとBを通じて顧客・社会に価値を届ける役割だと理解している。私は△△の経験を通じて、◇◇力を培い、課題を構造化して解決策を形にすることにやりがいを感じた。貴社の当コースではCに取り組めるため、自分の強みを活かしながら専門性を高められると考える。将来は〇〇として、□□の実現に貢献したい。
```

### 4-4. `300〜400字`

職種選択の理由を二つに分けることができます。「役割への魅力」と「自己適性」の二軸で書くと安定します。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 結論 | 10% | 志望理由 |
| 理由1 | 25% | 職種の役割への魅力 |
| 理由2 | 25% | 自己適性 |
| 具体業務 | 25% | 取り組みたいテーマ |
| 将来像 | 15% | 職種での貢献 |

**テンプレート**

```text
私が〇〇職を志望する理由は二つある。第一に、□□を実現できる役割に魅力を感じるからである。〇〇職は、AやBを通じて顧客課題を捉え、解決策を実装する仕事だと理解している。第二に、自身の◇◇力を活かせると考えるからである。私は△△の経験で、課題を構造化し、周囲を巻き込みながら成果を出す力を培った。貴社の当コースではCやDに取り組めるため、私の志向と強みを活かしながら成長できると考える。将来は〇〇として、□□の価値創出に貢献したい。
```

### 4-5. `400〜500字`

職種理解を深く示せます。職種の魅力を抽象的に語るだけでなく、その職種が直面する難しさや必要能力に触れると説得力が増します。

| 要素 | 配分目安 | 書くこと |
|---|---:|---|
| 結論 | 10% | 職種志望理由 |
| 役割理解 | 25% | 職種の役割・難しさ |
| 原体験 | 20% | なぜその仕事に関心があるか |
| 自己適性 | 20% | 強み・経験 |
| 具体業務 | 15% | 取り組みたいテーマ |
| 将来像 | 10% | 価値発揮 |

**テンプレート**

```text
私が〇〇職を志望する理由は、□□を通じて顧客・社会の課題解決に貢献したいからである。私はこの職種を、Aを起点に課題を捉え、BやCを組み合わせて解決策を形にする役割だと理解している。単に技術や知識を持つだけでなく、相手の状況を理解し、関係者と合意形成しながら価値を届ける力が求められる点に魅力を感じている。私は△△の経験で、◇◇という課題に対して原因を分解し、周囲を巻き込みながら改善を進めた。この経験から、複雑な課題に向き合い、解決まで推進することにやりがいを感じた。貴社の当コースではDやEに取り組めるため、自身の強みを活かしつつ専門性を高められると考える。将来は〇〇職として、□□の実現に貢献したい。
```

### 4-6. `500字以上`

500字以上では、職種理解・自己適性・将来像を段落ごとに分けます。複数テーマを書く場合は、羅列ではなく選定理由を一つずつ示します。

**推奨構成**

```text
第1段落：職種・コース志望理由の結論
第2段落：職種の役割理解
第3段落：関心が生まれた経験
第4段落：自分の強み・適性
第5段落：取り組みたい業務・テーマ
第6段落：将来の価値発揮
```

**テンプレート**

```text
私が〇〇職／〇〇コースを志望する理由は、□□を実現したいからである。

私はこの職種を、Aを通じて顧客や事業の課題を捉え、BやCを組み合わせて解決策を実装する役割だと理解している。特に、単に提案や開発を行うのではなく、相手の状況や制約を踏まえて価値を届ける点に、この職種の難しさと魅力があると考える。

この関心は、△△の経験から生まれた。当時、私はDという課題に直面し、Eを行うことでFという成果を得た。この経験から、課題を構造化し、周囲と協働しながら解決策を形にすることにやりがいを感じた。

私の強みは◇◇力である。この強みは、Gの経験で発揮され、Hという成果につながった。〇〇職では、この力を活かして、課題発見から実行まで粘り強く推進できると考えている。

当コースでは、特にIやJに取り組みたい。理由は、これらが私の関心である□□と直結し、かつ実務を通じてAの視点を学べると考えるためである。

将来は〇〇職として、専門性と顧客理解を兼ね備え、□□の価値創出に貢献したい。
```

## 5. 複数希望テーマを書く場合の型

```text
【第1希望】〇〇。理由は、△△の経験を活かし、□□に挑戦したいからだ。
【第2希望】〇〇。理由は、Aへの関心があり、Bの実装を学びたいからだ。
【第3希望】〇〇。理由は、Cの知見を深め、Dに応用したいからだ。
```

## 8. NGパターン

```text
貴社に魅力を感じたため、このコースを志望する。成長できる環境で学びたい。
```

弱い理由は、企業志望理由に寄りすぎていて、職種・コースの役割理解が見えないためです。

## 9. 最終チェック

- 企業志望理由ではなく、職種・コース志望理由になっているか
- その職種の役割を自分の言葉で説明できているか
- 自分の強み・経験が、その職種でどう活きるか明確か
- 具体的に取り組みたい業務やテーマがあるか
- 将来の価値発揮までつながっているか
