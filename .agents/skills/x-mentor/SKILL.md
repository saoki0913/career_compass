# X Mentor — バズるポスト作成スキル

X (Twitter) で圧倒的にリーチするポストを作成する専門スキル。書き出しのパターン提案、ターゲット別の言葉遣い調整、エンゲージメント最大化の構造修正まで一気通貫で行う。

## 起動トリガー

ユーザーが以下のいずれかを依頼したとき:
- X / Twitter のポスト作成・添削
- バズるツイートを書いてほしい
- SNS 発信の相談

## ワークフロー

### Step 1: ヒアリング（省略可）

ユーザーが十分な情報を提供済みなら省略する。不足時のみ確認:

- **伝えたい内容**: 何を伝えたいか（1行でOK）
- **ターゲット**: 誰に届けたいか（例: エンジニア、経営者、就活生、一般層）
- **目的**: 認知拡大 / 権威性構築 / 集客 / 共感獲得 / 議論喚起
- **トーン**: カジュアル / 権威的 / 煽り系 / 共感系 / 教育的

### Step 2: フック（書き出し）3パターン提案

読み手の指を止める書き出しを3パターン提案する。各パターンは異なるフック手法を使う:

| フック手法 | 例 |
|---|---|
| **逆張り** | 「〇〇は意味ない。本当に必要なのは——」 |
| **数字インパクト** | 「年収1億稼いだ人の共通点、たった1つだけ。」 |
| **共感→反転** | 「昔の自分に言いたい。〇〇するな。」 |
| **権威 + 秘密** | 「10年やってわかった、誰も教えてくれない〇〇の真実。」 |
| **疑問投げかけ** | 「なぜ〇〇する人ほど成功するのか？」 |
| **体験談** | 「〇〇したら人生変わった話。」 |
| **箇条書き予告** | 「〇〇に必要なこと5選（最後が一番大事）」 |

### Step 3: ターゲット別トーン調整

選ばれたフックをベースに、ターゲットに合わせて言葉遣いを最適化:

- **エンジニア向け**: 具体的・論理的・専門用語OK・過度な煽りNG
- **経営者/ビジネス層**: ROI・数字・意思決定フレーム・簡潔
- **就活生/学生**: 親しみやすい・不安に寄り添う・具体的行動指示
- **一般層**: 平易な言葉・感情に訴える・日常の例え
- **クリエイター**: 感性・ストーリー・ビジュアル喚起

### Step 4: 構造最適化

伸びやすい構造パターンに当てはめて最終調整:

#### 構造パターン

1. **フック → 本題 → CTA**（最もシンプル、汎用性高）
2. **フック → 箇条書き → まとめ**（情報系に強い）
3. **フック → ストーリー → 教訓**（共感・拡散向き）
4. **問いかけ → 回答 → 深掘り**（議論喚起向き）
5. **Before → After → How**（変化を見せたいとき）

#### チェックリスト

- [ ] 1行目で指が止まるか（フックの強度）
- [ ] 140文字以内で核心が伝わるか（TL表示の最適化）
- [ ] 「自分ごと」として読めるか（共感性）
- [ ] リプ・引用RTしたくなる余白があるか（エンゲージメント）
- [ ] 句読点・改行で読みやすいか（可読性）
- [ ] 炎上リスクはないか（安全性）

### Step 5: 最終出力

以下の形式で提出:

```
【ポスト本文】
（実際の投稿テキスト）

【フック手法】〇〇型
【ターゲット】〇〇
【構造】〇〇パターン
【想定エンゲージメント要因】（なぜ伸びそうか1行）
```

## 応用テクニック

### スレッド（連投）構成
1本目でフック → 2本目以降で深掘り → 最終で CTA or まとめ。
1本目だけで完結しても価値がある状態にする。

### リプライ誘導
「あなたはどっち派？」「これ共感する人RT」など、行動を促す一文を入れる。

### 時間帯の推奨
- 朝 7-9時: 通勤中の社会人
- 昼 12-13時: 休憩中の全年齢
- 夜 20-22時: リラックスタイム（最もエンゲージ高い）
- 深夜 0-1時: コアなフォロワー向け

## 禁止事項

- 誇大表現や虚偽の実績を書かない
- 他者を攻撃・中傷する内容にしない
- ステルスマーケティングに該当する表現をしない
- 著作権を侵害する引用をしない
