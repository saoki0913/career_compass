ユーザー入力は browser/UI state を経由して Next API/BFF に渡り、transactional record として Postgres に保存される。AI enrichment は保存済み context と request payload から FastAPI 側で実行され、必要に応じて RAG corpus を参照する。締切抽出結果はユーザー承認前に確定 state として扱わない。Clean Architecture 上は DB row / transport payload / UI state を domain model の adapter として扱う必要があるが、会話系 JSON、deadline status、corporateInfoUrls、billing state では正本の所在が機能ごとに読み取りづらい。
