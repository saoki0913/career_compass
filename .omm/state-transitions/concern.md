状態管理の最大リスクは、状態主体ごとに正本が明文化されていない箇所があること。Deadline は statusOverride / completedAt / tasks.status / autoCompletedTaskIds、Conversation は DB JSON / FastAPI payload / TS domain / hook state、RAG ingest は corporateInfoUrls / ingest jobs / vector metadata / status endpoint、Billing は Stripe webhook / subscription row / credits / feature policy が絡む。状態遷移を route や UI に追加し続けると、1 箇所の修正に必要な前提知識が急増する。
